# 🌐 [완벽 통합판] 15강. 링크 계층의 기초 및 MAC 프로토콜

## 1. 링크 계층(Link Layer)의 도입: "한 호흡의 전송"

네트워크 계층이 출발지 호스트에서 목적지 호스트까지의 전체 경로(End-to-End)를 담당했다면, **링크 계층은 물리적으로 직접 연결된 인접 노드 간의 데이터 전송**을 책임집니다.

*   **주요 용어 정리:**
    *   **노드(Nodes):** 호스트와 라우터.
    *   **링크(Links):** 인접 노드들을 연결하는 통신 채널 (유선, 무선, LAN 등).
    *   **프레임(Frame):** 링크 계층의 전송 단위로, 네트워크 계층의 데이터그램을 캡슐화한 것.
*   **핵심 비유:** 네트워크 계층까지가 "서울에서 부산까지 어떻게 갈지" 고민하는 단계였다면, 링크 계층은 "지금 당장 내 앞의 게이트웨이 라우터까지 어떻게 충돌 없이 보낼 것인가"라는 **'한 호흡(Hop)'**의 문제를 다룹니다.

---

## 2. 링크 계층은 어디에 존재하는가?

다른 상위 계층과 달리 링크 계층은 소프트웨어와 하드웨어의 경계에 위치합니다.

*   **구현 위치:** 각 호스트의 **네트워크 인터페이스 카드(NIC)** 또는 어댑터라 불리는 하드웨어 칩에 구현되어 있습니다.
*   **비교:** 전송 계층(TCP/UDP)과 네트워크 계층(IP)이 운영체제(OS) 내부에 소프트웨어 코드로 구현되는 것과 대조적입니다.
*   **기능:** 송신 측에서는 데이터그램을 프레임으로 캡슐화하고 에러 검출 비트를 추가하며, 수신 측에서는 에러를 확인하고 데이터그램을 추출해 상위 계층으로 올립니다.

---

## 3. 브로드캐스트 미디엄과 충돌(Collision)의 문제

우리가 사용하는 많은 링크(와이파이, 이더넷 등)는 여러 노드가 하나의 매체를 공유하는 **브로드캐스트(Broadcast)** 특성을 가집니다.

*   **공유의 비극:** 공기(무선)나 케이블(유선)이라는 매체를 공유하기 때문에, 내가 보낸 신호는 주변 모든 노드에 퍼집니다.
*   **충돌(Collision):** 만약 두 명 이상의 노드가 동시에 신호를 보내면 신호가 섞여서 '쓰레기 데이터'가 되고, 수신자는 이를 해독할 수 없게 되어 채널 자원이 낭비됩니다.
*   **해결책(MAC):** 공유 매체에 대한 접근을 조절하여 충돌을 방지하거나 해결하는 기술을 **매체 접근 제어(MAC: Medium Access Control)**라고 부릅니다.
*   이더넷주소(MAC주소) : 48비트, 하드웨어에 고정, 전세계유일

---

## 4. MAC 프로토콜의 3가지 유형

### 4.1 채널 분할 방식 (Channel Partitioning)
공유 자원을 시간, 주파수 등으로 쪼개어 각 노드에게 전용 칸을 배정하는 방식입니다.
*   **TDMA (Time Division):** 시간을 슬롯으로 나누어 자기 차례에만 전송합니다.
*   **FDMA (Frequency Division):** 주파수 대역을 나누어 각자 지정된 주파수로만 전송합니다.
*   **장단점:** 충돌이 절대 나지 않아 사용자가 많을 때 유리하지만, 전송할 데이터가 없는 노드에게도 자원이 할당되어 자원 낭비가 발생합니다.

### 4.2 랜덤 액세스 방식 (Random Access)
정해진 순서 없이 보내고 싶을 때 즉시 전송하는 방식입니다. 현재 이더넷이나 와이파이에서 주로 사용합니다.
*   **CSMA (Carrier Sense Multiple Access):** "말하기 전에 듣기(Listen before talk)". 매체가 사용 중이면 기다리고, 비어 있으면 전송합니다.
*   **CSMA/CD (Collision Detection):** 전송 중에 충돌이 감지되면 즉시 전송을 중단하여 매체 낭비를 줄입니다. 주로 유선 이더넷에서 사용합니다.
*   **전송 중단 후 : Binary Exponential Backoff:** 
*   **장단점:** 사용자가 적을 때 효율적이고 빠르지만, 사용자가 많아지면 충돌이 빈번해지고 성능이 급격히 저하됩니다.

### 4.3 순서 교대 방식 (Taking Turns)
중재자를 두거나 토큰을 돌려 순서를 정하는 방식입니다. 많이 안쓰임
*   **폴링(Polling):** 마스터 노드가 각 노드에게 보낼 게 있는지 일일이 물어봅니다. master작동 멈추면 완전 망함
*   **토큰 패싱(Token Passing):** '토큰'이라는 특수한 제어권을 가진 노드만 전송할 수 있으며, 전송 후 옆 노드에게 토큰을 넘깁니다. 토큰을 잃어버리는 순간 망함
*   **장단점:** 충돌을 방지하면서도 자원 낭비를 줄이지만, 마스터 노드나 토큰이 사라지면 전체 네트워크가 마비되는 치명적인 결함이 있습니다.

---

## 5. 핵심 메커니즘: CSMA/CD와 이진 지수 백오프

**이더넷의 CSMA/CD 작동 원리**입니다.

1.  **감지:** 데이터를 보내기 전 채널이 비어 있는지 확인합니다.
2.  **전송 중 충돌 발생:** 빛의 속도 한계(전파 지연) 때문에, 채널이 비어 있다고 판단하고 동시에 전송을 시작하면 충돌이 발생할 수밖에 없습니다.
3.  **중단:** 충돌이 감지되면 즉시 전송을 멈추고 잼(Jam) 신호를 보내 다른 노드들에게도 알립니다.
4.  **이진 지수 백오프 (Binary Exponential Backoff):**
    *   충돌 후 즉시 재전송하면 또 충돌할 확률이 높으므로, 일정 시간 기다린 후 다시 시도합니다.
    *   **연속 충돌 시:** 기다리는 시간의 범위를 **2배씩 지수적으로** 늘립니다 ($0$부터 $2^n-1$ 사이의 값 중 랜덤 선택).
    *   **이유:** 충돌이 잦다는 것은 주변에 말하려는 사람이 많다는 뜻이므로, 더 넓은 범위에서 시간을 골라야 충돌 확률이 낮아지기 때문입니다.

# 🌐 16강. 링크 계층: 이더넷, MAC 주소 및 ARP 완벽 정리

## 1. 이더넷 (Ethernet) 개요
이더넷은 현재 유선 LAN 환경에서 가장 지배적인 기술입니다.

*   **지배적인 이유:** NIC(랜카드)가 약 20달러 정도로 매우 저렴하며, 토큰 링(Token LAN)이나 ATM 방식보다 구조가 단순하면서도 효과적입니다. 또한, 10 Mbps에서 시작해 현재 10 Gbps까지 속도 경쟁에서 살아남으며 계속 발전해 왔습니다.
*   **서브넷과 LAN:** 라우터를 거치지 않고 직접 접근 가능한 호스트들의 집합인 '서브넷'은 실제 물리적으로는 LAN(Local Area Network)으로 구성됩니다.

## 2. 이더넷의 물리적 토폴로지 (Topology)
이더넷의 연결 형태는 시간에 따라 진화해 왔습니다.

*   **Bus형 (과거):** 모든 노드가 하나의 동축 케이블에 연결된 형태로, 모든 노드가 동일한 **충돌 도메인(Collision Domain)**에 있어 서로 충돌이 잦았습니다.
*   **Star형 (현재):** 중앙에 **스위치(Switch)**라는 장비를 두고 각 노드가 연결된 형태입니다. 각 노드는 독립적인 이더넷 프로토콜을 실행하며, 스위치가 충돌이 발생하지 않도록 중재합니다.

## 3. 이더넷 프레임 구조 (Frame Structure)
네트워크 계층의 IP 패킷은 링크 계층에서 '프레임'의 데이터 부분에 캡슐화됩니다.

*   **Preamble (8바이트):** 수신자와 송신자의 클록 속도를 동기화하기 위한 7바이트의 `10101010` 패턴과 1바이트의 `10101011`로 구성됩니다.
*   **주소 (Source/Dest Address):** 각각 6바이트(48비트)의 MAC 주소가 들어갑니다.
*   **Type:** 상위 계층의 프로토콜(주로 IP)이 무엇인지 명시하여 수신 측에서 알맞은 곳으로 전달할 수 있게 합니다.
*   **CRC:** 에러 체크를 위한 필드입니다.

## 4. CSMA/CD 알고리즘과 미니멈 프레임 사이즈
이더넷은 공유 매체에서의 충돌을 해결하기 위해 **CSMA/CD** 방식을 사용합니다.

### 4.1 동작 원리
1.  **Carrier Sense:** 말하기 전에 듣습니다. 채널이 비어 있으면 전송을 시작합니다.
2.  **Collision Detection:** 전송 중에 다른 신호와 겹쳐 충돌이 감지되면 즉시 전송을 중단하고 잼(Jam) 신호를 보냅니다.
3.  **Exponential Backoff:** 충돌 후 랜덤한 시간만큼 기다린 후 재전송하며, 연속 충돌 시 대기 시간의 범위를 지수적으로 늘립니다.

### 4.2 64바이트의 비밀 (미니멈 프레임 사이즈)
이더넷은 최소 프레임 크기를 **64바이트**로 강제합니다.
*   **이유:** 패킷이 너무 짧으면, 전파 지연(Propagation Delay) 때문에 저 멀리서 발생한 충돌 신호가 나에게 돌아오기 전에 전송이 끝나버릴 수 있습니다.
*   **해결:** 전송을 충분히 길게(최소 64바이트) 유지하여, 충돌이 났을 때 반드시 송신자가 이를 감지(Detect)할 수 있도록 보장하는 것입니다. 데이터가 부족하면 의미 없는 패딩을 채워서라도 이 길이를 맞춥니다.

## 5. MAC 주소 (MAC Address)
IP 주소가 논리적인 주소라면, MAC 주소는 인터페이스 자체에 부여된 물리적인 주소입니다.

*   **특징:** 48비트 주소 체계이며, 공장에서 나올 때 랜카드(NIC) ROM에 구워져 나옵니다.
*   **비유:** 주민등록번호(맥 주소)는 평생 변하지 않는 고유 정보이지만, 주소지(IP 주소)는 이사(네트워크 이동)할 때마다 변할 수 있는 것과 같습니다.
*   **구조:** 앞의 24비트는 제조사 식별 번호, 뒤의 24비트는 제품 고유 일련번호입니다.

## 6. ARP (Address Resolution Protocol)
상대의 IP 주소는 알지만 실제 데이터를 보내기 위해 필요한 **MAC 주소를 모를 때** 사용하는 프로토콜입니다.

*   **ARP 테이블:** IP와 MAC 주소의 매핑 정보를 캐시(Cache) 형태로 저장하는 테이블입니다.
*   **동작 과정:**
    1.  테이블에 정보가 없으면 **ARP Request**를 LAN 전체에 **브로드캐스트**(모두에게 질문)합니다.
    2.  해당 IP를 가진 호스트가 자신의 MAC 주소를 담아 **ARP Reply**를 유니캐스트(직접 응답)로 보냅니다.
    3.  이 정보를 테이블에 기록(TTL 동안 유지)하고 프레임을 완성하여 전송합니다.

## 7. 종합: 데이터 전송의 전체 흐름 (A부터 구글까지)
패킷이 여러 라우터를 거쳐 목적지까지 가는 과정은 다음과 같습니다.

1.  **IP 계층:** 출발지(A)와 목적지(Google)의 **IP 주소는 절대 변하지 않습니다**.
2.  **링크 계층:** 매 홉(Hop)을 지날 때마다 프레임을 새로 만듭니다.
    *   **포워딩 테이블**을 참조해 넥스트 홉 IP를 찾습니다.
    *   **ARP 테이블**을 참조해 해당 IP의 MAC 주소를 알아내어 프레임의 목적지 주소로 기록합니다.
3.  **변화:** 결과적으로 패킷이 이동함에 따라 **프레임 헤더의 소스/목적지 MAC 주소만 계속해서 바뀌며** 배달이 완료됩니다.

# 🌐 [학습 정리] 17강. 링크 계층: 외부 네트워크 전송 및 스위치의 세계

## 1. 다른 LAN으로의 전송 (Routing to another LAN)
ARP가 같은 네트워크 안에서의 주소 찾기였다면, 이번에는 **라우터를 거쳐 다른 네트워크(서브넷)에 있는 목적지**로 패킷을 보내는 구체적인 과정을 살펴봅니다.

### 1.1 시나리오: 호스트 A가 다른 네트워크의 호스트 B에게 데이터 전송
*   **준비물:** A는 B의 IP 주소를 알고 있으며, 자신의 첫 번째 홉 라우터(R)의 IP와 MAC 주소도 알고 있다고 가정합니다.
*   **핵심 원칙:** 전송 과정에서 **IP 주소는 절대 변하지 않지만, MAC 주소는 매 홉(Hop)마다 바뀝니다**,.

### 1.2 단계별 전송 과정
1.  **A의 패킷 생성:** IP 헤더에는 `출발지: A의 IP`, `목적지: B의 IP`를 적습니다.
2.  **A의 프레임 캡슐화:** 이 패킷을 보낼 때, 목적지 MAC 주소는 최종 목적지인 B가 아니라 **라우터 R의 MAC 주소**를 적습니다. 왜냐하면 일단 라우터까지 가야 하기 때문입니다,.
3.  **라우터 R의 처리:** 프레임을 받은 라우터는 껍데기(링크 계층 헤더)를 까서 IP 패킷을 꺼냅니다. 포워딩 테이블을 보고 B로 가기 위한 다음 인터페이스를 결정합니다.
4.  **라우터 R의 재전송:** 라우터는 다시 새로운 프레임을 만듭니다. 이때 `출발지 MAC: R의 출력 인터페이스 MAC`, `목적지 MAC: B의 MAC 주소`를 적어 전송합니다,.

## 2. 스위치 (Switch): 현대 LAN의 핵심 장비
과거에는 하나의 선을 공유하는 버스(Bus) 형태를 썼지만, 요즘은 **스위치**를 중심으로 한 별(Star) 모양 구조를 사용합니다,.

### 2.1 스위치의 특징
*   **충돌 도메인 분리:** 스위치는 각 포트마다 독립적인 통신을 가능하게 하여 **충돌(Collision)을 방지**합니다. 예를 들어 A-A' 통신과 B-B' 통신이 동시에 일어나도 충돌하지 않습니다,.
*   **투명성(Transparent):** 호스트들은 중간에 스위치가 있는지조차 모릅니다. 스위치는 IP나 MAC 주소를 가지지 않는 **링크 계층 장치**이기 때문입니다,.
*   **플러그 앤 플레이:** 관리자가 별도로 설정할 필요 없이 꽂기만 하면 스스로 학습하며 작동합니다.

### 2.2 셀프 러닝 (Self-Learning)
스위치는 **스위치 테이블**을 가지고 있으며, 누가 어느 포트에 연결되어 있는지 스스로 배웁니다,.
1.  **학습:** 어떤 포트로 프레임이 들어오면, 그 프레임의 **출발지 MAC 주소**를 보고 "아, 이 주소는 이 포트에 있구나"라고 테이블에 기록합니다,.
2.  **전달(Forwarding):** 목적지 MAC 주소가 테이블에 있으면 해당 포트로만 쏙 보내줍니다,.
3.  **플러딩(Flooding):** 만약 목적지가 어디 있는지 모르면, 들어온 포트를 제외한 **모든 포트로 복사해서 뿌립니다**. 이후 목적지가 응답하면 그 위치를 다시 학습합니다,.

## 3. 스위치 vs 라우터 vs 가정용 공유기 (비교)

교수님은 질문 답변을 통해 우리가 흔히 쓰는 장비들의 차이점을 명확히 해주셨습니다,.

*   **라우터 (L3):** 네트워크 계층 장비. IP 주소를 보고 경로를 결정하며, 인터페이스마다 IP와 MAC 주소를 가집니다,.
*   **스위치 (L2):** 링크 계층 장비. MAC 주소만 보고 전달하며, IP 주소가 없습니다 (호스트 눈에 안 보임),.
*   **가정용 공유기:** 사실상 **라우터 + 스위치 + NAT + DHCP + DNS 서버** 기능이 합쳐진 작은 컴퓨터입니다,. 내부적으로 사설 IP를 할당하고 관리하는 복잡한 역할을 수행합니다.

## 4. 데이터 센터 네트워크 (Data Center Networks)
수만 대의 서버가 있는 데이터 센터는 일반적인 방식으로는 연결이 어렵습니다.

*   **계층적 구조:** 서버들을 랙(Rack) 단위로 묶고, 그 위에 **TOR(Top of Rack) 스위치**를 둡니다. 이를 다시 상위 스위치(Tier-2, Tier-1)들로 촘촘하게 연결하여 거대한 네트워크를 형성합니다,.
*   **로드 밸런서 (Load Balancer):** 외부 요청이 들어오면 수많은 서버 중 현재 가장 한가한 서버로 업무를 배분해 주는 역할을 합니다,.

## 5. 결론: 웹 요청의 하루 (A Day in the Life)
우리가 웹 브라우저에 `google.com`을 치면 일어나는 일들입니다,.
1.  **DHCP:** 내 컴퓨터가 IP 주소를 할당받습니다.
2.  **DNS:** 구글의 IP 주소를 알아내기 위해 UDP 패킷을 보냅니다.
3.  **ARP:** 게이트웨이 라우터의 MAC 주소를 알아냅니다.
4.  **TCP:** 구글 서버와 3-Way Handshake로 연결을 맺습니다.
5.  **HTTP:** 드디어 GET 요청을 보내고 웹 페이지를 받아옵니다.
