# [학습 정리] 컴퓨터 네트워크의 기초 (Introduction)


## 1. 인터넷(Internet)이란 무엇인가?
인터넷은 전 세계적으로 수십억 개의 컴퓨팅 장치가 연결된 **네트워크의 네트워크(network of networks)**입니다,.

*   **호스트(Hosts) / 엔드 시스템(End Systems):** 네트워크의 가장자리에 위치하여 애플리케이션 프로그램을 실행하는 장치들(PC, 서버, 스마트폰 등)을 의미합니다.
*   **통신 링크(Communication Links):** 데이터가 전송되는 물리적 매체로, 광섬유, 구리선, 무선 라디오 스펙트럼 등이 있으며 전송 속도는 **대역폭(Bandwidth, bps)**으로 측정됩니다.
*   **패킷 스위치(Packet Switches):** 데이터를 '패킷(Packet)'이라는 덩어리로 전달하는 장치로, 대표적으로 **라우터(Router)**와 **스위치(Switch)**가 있습니다.

---

## 2. 프로토콜 (Protocol)
네트워크에서 모든 통신 활동은 프로토콜에 의해 제어됩니다. 프로토콜은 둘 이상의 통신 개체 간에 **교환되는 메시지의 형식(Format), 순서(Order), 그리고 메시지 전송 및 수신 시 취해야 할 행동**을 정의합니다.

*   **인간 프로토콜의 예:** "안녕하세요?"라고 묻고, 상대방이 응답하면 시간을 물어보는 상호작용.
*   **네트워크 프로토콜의 예:** TCP 연결 요청 → TCP 연결 응답 → HTTP Get(파일 요청) → 데이터 전송.

---

## 3. 네트워크 구조 (Network Structure)
네트워크는 크게 세 부분으로 나뉩니다:
1.  **Network Edge (네트워크 가장자리):** 애플리케이션과 호스트가 위치한 곳.
2.  **Network Core (네트워크 코어):** 라우터들이 서로 연결되어 데이터를 전달하는 그물망 형태의 네트워크,.
3.  **Access Networks, Physical Media (접세스 네트워크 및 물리 매체):** 엔드 시스템을 코어 라우터에 연결하는 통신 링크.

### 3.1 Network Edge: 서비스 모델
*   **Client/Server 모델:** 클라이언트(PC, 브라우저)가 항상 켜져 있는 서버에 서비스를 요청하고 받는 구조입니다.
*   **Peer-to-Peer (P2P) 모델:** 전용 서버의 사용을 최소화하고, 엔드 시스템끼리 직접 통신하는 구조입니다 (예: Skype, BitTorrent).

### 3.2 연결 지향형 서비스: TCP (Transmission Control Protocol)
데이터를 전송하기 전, 두 호스트 간에 연결 상태를 설정하는 과정이 필요합니다.
*   **신뢰성(Reliability):** 데이터가 순서대로, 손실 없이 전달됨을 보장합니다.
*   **흐름 제어(Flow Control):** 송신자가 수신자의 처리 능력을 초과하여 데이터를 보내지 않도록 조절합니다.
*   **혼잡 제어(Congestion Control):** 네트워크가 혼잡할 때 송신 속도를 줄여 네트워크 붕괴를 막습니다.

---

## 4. 네트워크 코어 (Network Core)
데이터가 네트워크를 통해 전달되는 방식은 크게 두 가지로 나뉩니다.

### 4.1 회선 교환 (Circuit Switching)
*   **특징:** 통신 경로상에 필요한 자원(대역폭)을 **미리 예약**합니다.
*   **장점:** 자원을 독점하므로 성능이 보장됩니다 (Guaranteed performance).
*   **단점:** 통신 중이 아닐 때도 자원을 점유하므로 효율성이 낮고 연결 설정 시간이 필요합니다.
*   **비유:** 예약제로 운영되는 식당.

### 4.2 패킷 교환 (Packet Switching)
*   **특징:** 데이터를 **패킷**이라는 작은 단위로 나누어 전송하며, 자원을 미리 예약하지 않고 필요할 때 사용합니다.
*   **통계적 다중화(Statistical Multiplexing):** 여러 사용자가 수요에 따라 동적으로 링크 자원을 공유하는 방식입니다.
*   **효율성 비교:** 1Mbps 링크에서 100kbps를 사용하는 사용자가 있을 때, 회선 교환은 10명만 수용 가능하지만, 패킷 교환은 더 많은 사용자를 수용할 수 있습니다 (사용자가 항상 활동하는 것은 아니기 때문).

---

## 5. 패킷 교환 네트워크에서의 지연(Delay)
데이터 패킷이 하나의 노드(라우터)를 통과할 때 발생하는 총 지연 시간은 다음 4가지 요소의 합입니다,.

$$d_{nodal} = d_{proc} + d_{queue} + d_{trans} + d_{prop}$$

1.  **Nodal Processing (노드 처리 지연):** 비트 오류 확인 및 출력 링크 결정에 걸리는 시간.
2.  **Queueing (큐잉 지연):** 출력 링크가 사용 중일 때 큐(대기열)에서 기다리는 시간. 네트워크 혼잡도에 따라 달라집니다.
3.  **Transmission Delay (전송 지연):** 패킷의 모든 비트를 링크로 밀어내는 데 걸리는 시간.
    *   $L$ (패킷 길이, bits), $R$ (링크 대역폭, bps) 일 때, $d_{trans} = L/R$
4.  **Propagation Delay (전파 지연):** 링크의 시작점에서 다음 라우터까지 물리적으로 신호가 이동하는 시간.
    *   $d$ (링크 길이), $s$ (전파 속도, $~2 \times 10^8$ m/sec) 일 때, $d_{prop} = d/s$

> **중요 차이점:** 전송 지연($L/R$)은 송신 장치가 데이터를 내보내는 속도와 관련이 있고, 전파 지연(d/s)은 매체를 통해 신호가 이동하는 속도와 관련이 있습니다.

---

## 6. 보완 설명 및 이해를 돕는 요약
*   **인터넷 구조의 복잡성:** 인터넷은 단순히 연결된 선들의 모임이 아니라, 수많은 ISP(Internet Service Provider)들이 계층적으로 얽혀 있는 거대한 구조입니다.
*   **패킷 손실(Loss):** 라우터의 버퍼(큐) 용량은 유한합니다. 패킷이 도착했을 때 큐가 가득 차 있다면 해당 패킷은 버려지게 되는데, 이것이 패킷 손실입니다.
