# 🌐 이석복 컴퓨터네트워크 핵심 개념 요약

---

## 1. 네트워크 개요

### 인터넷 구조
- **Edge**: 호스트(클라이언트·서버), 엔드시스템
- **Core**: 라우터들의 망 — 패킷 스위칭으로 데이터 전달
- **Access Network**: DSL, Cable, Fiber, WiFi, 4G/5G

### 패킷 스위칭 vs 회선 스위칭

| | 패킷 스위칭 | 회선 스위칭 |
|---|---|---|
| 자원 할당 | 필요할 때만 | 연결 시 미리 예약 |
| 효율 | 높음 (통계적 다중화) | 낮음 |
| 지연 | 큐잉 딜레이 발생 가능 | 일정 |
| 사용 | 인터넷 | 전화망 |

### 딜레이 4종류
```
d_nodal = d_proc + d_queue + d_trans + d_prop

d_proc  : 처리 지연 (패킷 헤더 검사, 라우팅 테이블 조회)
d_queue : 큐잉 지연 (트래픽 몰릴 때 대기)
d_trans : 전송 지연 = L(bits) / R(bps)  ← 패킷 크기·대역폭
d_prop  : 전파 지연 = d(거리) / s(빛속도)  ← 물리적 거리
```

**처리량(Throughput)**: `min(R_s, R_c, R_bottleneck)` — 병목 링크가 결정

### 프로토콜 계층 (5계층)

| 계층 | 역할 | PDU |
|------|------|-----|
| Application | HTTP, DNS, SMTP | Message |
| Transport | TCP, UDP — 프로세스 간 통신 | Segment |
| Network | IP — 호스트 간 라우팅 | Datagram |
| Link | Ethernet, WiFi — 인접 노드 간 | Frame |
| Physical | 비트 전송 | Bit |

## 2. Application Layer

### HTTP
- TCP 기반, **Stateless** (상태 저장 안 함)
- **Non-persistent**: 요청마다 새 TCP 연결 → 2 RTT + 전송시간
- **Persistent**: 연결 유지 → 1 RTT + 전송시간 (HTTP/1.1 기본)
- **Pipelining**: 응답 기다리지 않고 여러 요청 연속 발송

### HTTP 메서드·상태코드
```
GET    : 리소스 요청
POST   : 데이터 전송
PUT    : 리소스 수정
DELETE : 리소스 삭제

200 OK
301 Moved Permanently
304 Not Modified
400 Bad Request
404 Not Found
505 HTTP Version Not Supported
```

### 쿠키
- 상태 없는 HTTP를 상태 있게 만드는 방법
- 서버 → 클라이언트: `Set-cookie` 헤더
- 클라이언트 → 서버: `cookie` 헤더 (이후 요청에 포함)

### 웹 캐시 (Proxy Server)
- 원서버 대신 응답 → 응답시간↓, 트래픽↓
- **조건부 GET**: `If-modified-since` 헤더로 캐시 유효성 확인
  - 변경 없으면 → `304 Not Modified` (body 없음)
  - 변경 있으면 → `200 OK` + 새 데이터

### DNS (Domain Name System)
- 분산·계층적 데이터베이스
- **계층 구조**: Root NS → TLD NS (.com, .kr) → Authoritative NS
- **Local DNS**: ISP가 제공, 먼저 캐시 확인

**질의 방식**

| 방식 | 설명 |
|------|------|
| Recursive | 하위 서버가 직접 최종 답을 찾아 반환 |
| Iterative | 각 서버가 다음 서버 주소만 알려줌 (실제 인터넷 방식) |

**DNS 레코드 타입**

| 타입 | 의미 |
|------|------|
| A | 호스트명 → IPv4 주소 |
| AAAA | 호스트명 → IPv6 주소 |
| CNAME | 별칭 → 정식 호스트명 |
| MX | 메일 서버 주소 |
| NS | 해당 도메인의 네임서버 |

### SMTP (이메일)
- TCP 25번 포트, Push 프로토콜
- **3단계**: Handshaking → 메시지 전송 → 종료
- HTTP는 Pull(클라이언트가 요청), SMTP는 Push(서버가 전송)
- 수신: POP3 / IMAP / HTTP (웹메일)

## 3. Transport Layer

### UDP
- **비연결형**, 비신뢰적 전송
- 헤더: Source Port | Dest Port | Length | Checksum (8바이트)
- 사용: DNS, DHCP, 스트리밍, VoIP
- 장점: 빠름, 연결 설정 오버헤드 없음

### TCP
- **연결형**, 신뢰적, 순서 보장, 흐름제어, 혼잡제어
- **3-way Handshake**: SYN → SYN-ACK → ACK
- **4-way Handshake (종료)**: FIN → ACK → FIN → ACK

### TCP 신뢰적 데이터 전송

**재전송 메커니즘**
```
- Timeout 발생 시 재전송
- Duplicate ACK 3개 시 빠른 재전송 (Fast Retransmit)
```

**Go-Back-N vs Selective Repeat**

| | Go-Back-N | Selective Repeat |
|---|---|---|
| 오류 발생 시 | 해당 패킷부터 전부 재전송 | 오류 패킷만 재전송 |
| 수신 버퍼 | 없음 (순서 어긋나면 버림) | 있음 (저장해둠) |
| 윈도우 크기 | N-1 이하 | N/2 이하 |

### TCP 흐름 제어 (Flow Control)
- **수신 윈도우(rwnd)**: 수신 버퍼 남은 공간을 송신자에게 알림
- 송신자는 `min(cwnd, rwnd)` 이하로 전송
- 목적: **수신자 버퍼 오버플로우 방지**

### TCP 혼잡 제어 (Congestion Control)
- 목적: **네트워크 혼잡 방지** (라우터 버퍼 오버플로우)

**단계별 동작**

| 단계 | 조건 | 동작 |
|------|------|------|
| Slow Start | 시작 / timeout 후 | cwnd 1 MSS → 매 RTT 2배 증가 |
| Congestion Avoidance | cwnd ≥ ssthresh | 매 RTT 1 MSS 선형 증가 |
| Fast Recovery | 3 dup ACK | ssthresh = cwnd/2, cwnd = ssthresh + 3 |
| Timeout 발생 | — | ssthresh = cwnd/2, cwnd = 1 (Slow Start 재시작) |

**TCP Tahoe vs Reno**

| 이벤트 | Tahoe | Reno |
|--------|-------|------|
| Timeout | cwnd=1, Slow Start | cwnd=1, Slow Start |
| 3 dup ACK | cwnd=1, Slow Start | cwnd=ssthresh, Fast Recovery |

### 포트 번호

| 프로토콜 | 포트 |
|---------|------|
| HTTP | 80 |
| HTTPS | 443 |
| FTP | 21 |
| SSH | 22 |
| DNS | 53 |
| SMTP | 25 |

## 4. Network Layer

### IP (IPv4)

**IP 주소 구조**
```
32비트 = 네트워크부 + 호스트부
CIDR 표기: 192.168.1.0/24
서브넷 마스크: 255.255.255.0
```

**특수 주소**
```
127.0.0.1      : Loopback (자기 자신)
0.0.0.0        : 모든 주소 (default route)
255.255.255.255: 브로드캐스트
```

### DHCP (Dynamic Host Configuration Protocol)
- UDP 기반, 67/68번 포트
- **4단계**: Discover → Offer → Request → ACK
- IP 주소를 동적으로 할당, 임대 시간(Lease Time) 존재

### NAT (Network Address Translation)
- 사설 IP → 공인 IP 변환 (포트 번호로 구분)
- IPv4 주소 고갈 문제 완화
- NAT 테이블로 내부·외부 매핑 관리

### IPv6
- 128비트 주소 (32비트 IPv4의 4배)
- 헤더 단순화 (체크섬 제거, 옵션 → Extension Header)
- IPv4 → IPv6 전환: **터널링** (IPv6 패킷을 IPv4로 캡슐화)

### 라우팅 알고리즘

**Link State (LS)**
- 전체 토폴로지 파악 후 최단경로 계산
- 알고리즘: **다익스트라(Dijkstra)**
- 프로토콜: OSPF

**Distance Vector (DV)**
- 이웃에게 자신의 거리 벡터만 공유
- 알고리즘: **벨만-포드(Bellman-Ford)**
- 프로토콜: RIP
- 문제: **Count-to-Infinity** (라우팅 루프)

### 라우팅 프로토콜

| 프로토콜 | 타입 | 알고리즘 | 영역 |
|---------|------|---------|------|
| OSPF | LS | 다익스트라 | AS 내부 (IGP) |
| RIP | DV | 벨만-포드 | AS 내부 (IGP) |
| BGP | Path Vector | — | AS 간 (EGP) |

**BGP**: 인터넷의 실질적 라우팅 프로토콜
- eBGP: 서로 다른 AS 간 경로 정보 교환
- iBGP: 같은 AS 내부 라우터 간 경로 정보 공유

## 5. Link Layer

### 오류 검출

| 방법 | 설명 | 수준 |
|------|------|------|
| Parity bit | 1비트 오류 검출 | 낮음 |
| Checksum | 합산 방식 (TCP/IP) | 중간 |
| CRC | 다항식 나눗셈 | 높음 (링크 계층) |

### 다중 접속 프로토콜 (MAC)

**채널 분할**
- TDMA: 시간 슬롯 분할
- FDMA: 주파수 분할

**랜덤 접속**
- **ALOHA**: 충돌 시 랜덤 대기 후 재전송
- **CSMA**: 전송 전 채널 감지
- **CSMA/CD**: 충돌 감지 시 즉시 중단 → 재전송 (유선 Ethernet)
- **CSMA/CA**: 충돌 회피 (무선 WiFi) — 충돌 감지 불가라서 CA 사용

### Ethernet

- 비연결형, 비신뢰적 (오류 복구는 상위 계층 담당)
- **MAC 주소**: 48비트, NIC에 하드웨어 고정
- **프레임 구조**: Preamble | Dest MAC | Src MAC | Type | Data | CRC

### 스위치 vs 라우터

| | 스위치 | 라우터 |
|---|---|---|
| 계층 | 2계층 (Link) | 3계층 (Network) |
| 기준 | MAC 주소 | IP 주소 |
| 영역 | LAN 내부 | LAN 간 연결 |
| 테이블 | MAC 테이블 (자동 학습) | 라우팅 테이블 |

### ARP (Address Resolution Protocol)
- **IP 주소 → MAC 주소** 변환
- 동일 서브넷: ARP 요청(브로드캐스트) → ARP 응답(유니캐스트)
- 다른 서브넷: 게이트웨이(라우터) MAC 주소로 전송

## 6. Wireless (무선)

### WiFi (IEEE 802.11)
- **CSMA/CA** 사용 (충돌 감지 불가 → 회피 방식)
- **ACK**: 무선은 오류가 많아서 링크 계층에서도 ACK 확인
- **RTS/CTS**: 숨겨진 노드 문제 해결 (Hidden Terminal Problem)

**숨겨진 노드 문제 (Hidden Terminal)**
```
A --- AP --- B
A와 B는 서로 보이지 않음
→ A, B 동시에 AP에 전송 → 충돌
→ RTS/CTS로 해결: 전송 전 예약
```

**노출된 노드 문제 (Exposed Terminal)**
```
A --- B --- C --- D
B가 A에게 전송 중일 때 C는 D에게 전송 가능
But C는 채널 사용 중으로 오해 → 불필요한 대기
```

### 이동성 (Mobility)

**핸드오프(Handoff)**: 이동 중 기지국(AP) 변경
- **간접 라우팅**: 홈 에이전트 경유 → 오버헤드 있지만 단순
- **직접 라우팅**: 현재 위치로 직접 전송 → 효율적, 복잡

**Mobile IP 개념**
```
홈 주소(Home Address): 영구 IP
CoA(Care-of-Address): 현재 위치 임시 IP
홈 에이전트: 홈 네트워크에서 패킷을 CoA로 터널링
```

### 4G/LTE 구조
- **UE**: 사용자 단말
- **eNodeB**: 기지국 (Base Station)
- **EPC**: 코어 네트워크 (인터넷 연결)
- LTE는 IP 기반 All-IP 네트워크

## 7. 멀티미디어 네트워킹

### 스트리밍 방식

| 방식 | 특징 |
|------|------|
| UDP 스트리밍 | 실시간, 손실 허용, 빠름 |
| HTTP 스트리밍 | TCP 기반, 버퍼링, 안정적 |
| DASH | 네트워크 상태에 따라 화질 자동 조절 (Netflix, YouTube) |

### DASH (Dynamic Adaptive Streaming over HTTP)
- 비디오를 여러 화질로 인코딩 → 청크 단위 저장
- 클라이언트가 대역폭 측정 후 화질 선택
- CDN(Content Delivery Network) 활용

### CDN (Content Delivery Network)
- 전 세계 분산 서버에 콘텐츠 복제
- 사용자와 가까운 서버에서 제공 → 지연↓
- DNS를 이용해 가장 가까운 CDN 서버로 유도

### VoIP (실시간 음성 통신)
- **지연 허용치**: 150ms 이하 (체감 기준)
- 패킷 손실 허용 (재전송보다 빠른 전달이 중요)
- **지터(Jitter)**: 패킷 도착 시간 불규칙
  - 해결: **플레이아웃 버퍼** (일정량 모은 후 재생)

### QoS (Quality of Service)

**문제**: 최선형(Best-effort) 인터넷은 지연·손실 보장 없음

**해결 메커니즘**

| 기법 | 설명 |
|------|------|
| 스케줄링 | WFQ(가중치 공정 큐잉)로 우선순위 차등 |
| 폴리싱 | 토큰 버킷으로 트래픽 속도 제한 |
| 표시(Marking) | 패킷에 우선순위 표시 (DiffServ) |

## 🔑 면접 자주 나오는 키워드 정리

### TCP vs UDP

| | TCP | UDP |
|---|---|---|
| 연결 | 연결 지향 | 비연결 |
| 신뢰성 | 보장 | 비보장 |
| 순서 | 보장 | 비보장 |
| 속도 | 느림 | 빠름 |
| 사용 | HTTP, FTP, SSH | DNS, 스트리밍, VoIP |

### OSI 7계층 vs TCP/IP 5계층

| OSI | TCP/IP |
|-----|--------|
| Application | Application |
| Presentation | ↑ |
| Session | ↑ |
| Transport | Transport |
| Network | Network |
| Data Link | Link |
| Physical | Physical |

### 자주 묻는 질문 핵심 답변

**Q. 3-way Handshake란?**
> TCP 연결 설정 과정. SYN → SYN-ACK → ACK. 양방향 통신 가능 여부와 초기 시퀀스 번호를 교환.

**Q. HTTP와 HTTPS 차이?**
> HTTP는 평문 전송, HTTPS는 TLS(SSL) 암호화 추가. 443 포트 사용.

**Q. 공인 IP와 사설 IP 차이?**
> 공인 IP는 인터넷에서 유일, 사설 IP는 내부 네트워크에서만 사용. NAT가 변환 담당.

**Q. DNS 동작 과정?**
> 로컬 캐시 확인 → Local DNS → Root NS → TLD NS → Authoritative NS 순서로 IP 반환.

**Q. 라우터와 스위치 차이?**
> 스위치는 2계층(MAC 주소 기반, LAN 내부), 라우터는 3계층(IP 주소 기반, LAN 간 연결).

**Q. CSMA/CD vs CSMA/CA?**
> CD(Collision Detection): 유선 Ethernet, 충돌 감지 후 중단·재전송. CA(Collision Avoidance): 무선 WiFi, 충돌 회피 방식(RTS/CTS).
