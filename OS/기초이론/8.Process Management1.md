1. 프로세스 생성 (Process Creation)
부모 프로세스(Parent process)와 자식 프로세스(Children process):

    COW기법(Copy-On-Write) : Write가 발생하면 Copy하겠다. 그전까지는 부모꺼 가지고만 있음.

    운영체제에서 프로세스는 독립적으로 무작위로 생겨나는 것이 아니라, 기존의 프로세스(부모)가 새로운 프로세스(자식)를 생성합니다.

    이로 인해 프로세스들은 트리(일촌 구조) 형태의 계층 구조를 형성하게 됩니다.

자원 공유 (Resource Sharing Options): 프로세스가 생성될 때 자원을 어떻게 나눌지에 대한 모델입니다.

    모두 공유: 부모와 자식이 모든 자원을 공유하는 모델

    일부 공유: 일부 자원만 공유하는 모델

    전혀 공유하지 않음: 부모와 자식이 자원을 전혀 공유하지 않는 모델 (일반적인 운영체제의 기본 원칙)

수행 (Execution):

    공존 모델: 부모와 자식이 동시에 실행되며 경쟁하는 모델

    대기 모델: 자식이 종료(terminate)될 때까지 부모가 기다리는(wait,blocked 상태) 모델

2. 프로세스 주소 공간 (Address Space)과 생성 과정
    주소 공간 복사: * 자식 프로세스는 부모 프로세스의 주소 공간(Code, Data, Stack)을 그대로 복사하여 생성됩니다. (예: UNIX의 fork() 시스템 콜)

    새로운 프로그램 적재:

        복사된 상태에서 그대로 머무르는 것이 아니라, 자식 프로세스 공간 위에 새로운 프로그램의 코드를 덮어씌워 실행하게 됩니다. (예: UNIX의 exec() 시스템 콜)

3. 주요 시스템 콜 (System Calls for Process Management)
UNIX/Linux 환경에서 프로세스 관리를 위해 제공하는 대표적인 시스템 콜 인터페이스입니다.

    fork(): 부모 프로세스를 복사하여 새로운 자식 프로세스를 생성합니다. 주소 공간과 CPU 문맥(Registers, Program Counter 등)이 그대로 복사됩니다.

    exec(): 기존 프로세스의 주소 공간을 새로운 프로그램으로 완전히 대체하여, 새로운 프로그램을 처음부터 실행하도록 만듭니다.

    wait(): 자식 프로세스가 종료될 때까지 부모 프로세스를 Block 상태(대기 상태)로 만듭니다. 자식이 종료되면 부모가 깨어나서 다음 코드를 수행합니다.

    exit(): 프로세스를 자발적으로 종료하고, 부모 프로세스에게 종료되었음을 알리는 메시지를 전달하며 자원을 반납합니다.

4. 프로세스 종료 (Process Termination)
    자발적 종료 (exit):

        프로세스가 마지막 명령을 수행한 뒤, 시스템 콜을 통해 운영체제에 종료를 알립니다.

        자식 프로세스는 부모 프로세스에게 상태 데이터를 보냅니다 (via wait).

        운영체제는 프로세스가 가졌던 모든 자원(메모리, 파일 등)을 회수합니다.

    비자발적 강제 종료 (abort): 부모 프로세스가 자식 프로세스의 수행을 강제로 종료시키는 경우입니다.

        자식 프로세스가 할당된 자원의 한계를 넘어선 과도한 자원을 요구할 때

        자식 프로세스에게 할당된 작업이 더 이상 필요하지 않을 때

        부모 프로세스가 종료(exit)될 때 (일반적인 OS 구조에서는 부모가 종료되면 자식들이 먼저 연쇄적으로 종료된 후 부모가 종료됩니다.)

프로세스 세계에서는 자식이 부모보다 먼저 죽어야함!!!!
