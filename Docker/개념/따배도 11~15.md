# [따배도] 4장. 도커 컨테이너 만들어보기

## 1. [이론] 컨테이너 빌드 전략과 Dockerfile
컨테이너를 만든다는 것은 애플리케이션 실행에 필요한 모든 요소를 하나로 패키징하는 과정입니다.

*   **무엇을 컨테이너로 만드는가?**:
    *   **운영 환경**: 애플리케이션이 동작할 기반(예: Node.js, Python, Java 등).
    *   **소스 코드**: 실제 실행될 프로그램 코드.
    *   **실행 명령어**: 컨테이너가 시작될 때 자동으로 동작할 명령.
*   **마이크로서비스 아키텍처(MSA)**: 다양한 언어(Polyglot)로 작성된 서비스(쇼핑 카트, 주문, 고객 관리 등)를 각각의 컨테이너로 만들어 독립적으로 운영하고 배포합니다.
*   **Dockerfile의 특징**:
    *   컨테이너를 빌드하기 위한 명령어들의 집합인 텍스트 파일입니다.
    *   **탑다운(Top-down)** 방식으로 한 줄씩 해석하며 이미지를 생성합니다.
*   **주요 지시어(Directives)**:
    *   `FROM`: 베이스 이미지 지정 (가장 먼저 작성 필수).
    *   `COPY` / `ADD`: 호스트 파일을 컨테이너 내부로 복사 (ADD는 압축 해제 및 URL 다운로드 기능 포함).
    *   `RUN`: 이미지 빌드 시점에 패키지 설치 등 명령 실행.
    *   `CMD` / `ENTRYPOINT`: 컨테이너 실행 시 동작할 명령 (CMD는 사용자 인자에 의해 덮어쓰기 가능).
    *   `ENV` / `WORKDIR` / `USER`: 환경 변수 설정, 작업 디렉토리 지정, 실행 사용자 변경(보안 강화).

## 2. [실습] 실제 컨테이너 생성 및 배포 과정
두 가지 예제(Node.js 앱과 우분투 기반 웹 서버)를 통해 실습을 진행합니다.

### 예제 1: Node.js 기반 헬로 월드 컨테이너
1.  **소스 준비**: 8080 포트로 접속하면 호스트 이름을 출력하는 `hello.js` 작성.
2.  **Dockerfile 작성**:
    ```dockerfile
    FROM node:12
    COPY hello.js /
    CMD ["node", "/hello.js"]
    ```
3.  **빌드**: `docker build -t hellojs:latest .` 명령 실행 (마지막 `.`은 현재 디렉토리를 의미).

### 예제 2: 우분투 기반 아파치 웹 서버 컨테이너
1.  **Dockerfile 작성**: 베이스 이미지를 `ubuntu:18.04`로 지정하고, 레이어 수를 줄이기 위해 `&&`를 사용해 패키지 설치 명령을 하나로 묶습니다.
2.  **포트 및 실행 설정**: `EXPOSE 80`으로 포트를 알리고, 아파치 서비스를 포그라운드(Foreground) 모드로 실행하도록 설정합니다.

### 컨테이너 배포 (Docker Hub)
1.  **로그인**: `docker login` 명령으로 인증을 수행합니다.
2.  **태그 설정**: 개인 저장소에 올리기 위해 `아이디/이미지명:태그` 형식으로 이름을 변경합니다 (예: `smlinux/webserver:v1`).
3.  **푸시**: `docker push` 명령을 사용해 도커 허브로 이미지를 업로드합니다.

# [따배도] 5장. 컨테이너 보관 창고 (Docker Registry)

## 1. [이론] 도커 레지스트리의 이해
도커 레지스트리는 컨테이너 이미지를 저장하고 관리하는 저장소입니다.

*   **종류**:
    *   **공용 레지스트리 (Public Registry)**: 대표적으로 **도커 허브(Docker Hub)**가 있으며, 10만 개 이상의 이미지가 저장되어 있습니다.
    *   **사설 레지스트리 (Private Registry)**: 보안이나 네트워크 환경을 고려하여 사내에 직접 구축해 운영하는 독립된 저장소입니다.
*   **이미지 구분**: 도커 허브에는 도커가 직접 관리하는 **오피셜 이미지(Official Image)**와 벤더사가 제공하는 이미지, 그리고 일반 사용자가 올린 이미지가 공존합니다.

## 2. [실습] 도커 허브 및 사설 레지스트리 운영
*   **도커 허브 활용**: 
    *   `docker login`을 통해 인증 후, `docker push` 명령어로 본인의 저장소(Repository)에 이미지를 업로드할 수 있습니다.
    *   업로드 전 반드시 `계정명/이미지명:태그` 형식으로 이미지 태그를 변경해야 합니다.
*   **사설 레지스트리 구축**: 
    *   `registry`라는 이름의 컨테이너를 실행하는 것만으로도 사내 저장소를 만들 수 있습니다.
    *   `docker run -d -p 5000:5000 --restart always --name registry registry:2` 명령을 사용합니다.
    *   사설 저장소에 올릴 때는 이미지 이름 앞에 `호스트IP:포트` 정보를 추가해야 합니다.

# [따배도] 6장. 도커 컨테이너 사용하기 (생명주기 관리)

## 1. [이론] 컨테이너 생명주기와 명령어
컨테이너는 **이미지 다운로드 → 생성 → 실행 → 관리 → 종료/삭제**의 흐름을 가집니다.

*   **이미지 관리**: `search`(검색), `pull`(다운로드), `images`(목록 확인), `inspect`(상세 정보), `rmi`(삭제) 명령어를 사용합니다.
*   **컨테이너 실행**:
    *   `create` + `start`: 컨테이너를 만들고 별도로 실행합니다.
    *   `run`: 다운로드, 생성, 실행을 한 번에 처리하며 가장 많이 쓰입니다.
*   **운영 및 모니터링**: `ps`(목록), `top`(프로세스 확인), `logs`(로그 확인), `inspect`(IP 등 상세 정보)를 활용합니다.

## 2. [실습] 컨테이너 운영 시나리오
*   **포트 및 접속 확인**: `inspect` 명령어로 컨테이너의 내부 IP를 확인한 후, `curl` 명령어를 통해 서비스가 정상 동작하는지 테스트합니다.
*   **내부 접속 및 수정**: `docker exec -it [컨테이너명] /bin/bash`를 사용하여 실행 중인 컨테이너 내부에 접속하고 파일을 수정할 수 있습니다.
*   **로그 실시간 모니터링**: `docker logs -f` 옵션을 사용하면 클라이언트의 접속 로그를 실시간으로 확인할 수 있습니다.
*   **강제 삭제**: 실행 중인 컨테이너는 바로 삭제할 수 없으나, `-f` 옵션을 사용하면 중지와 삭제를 동시에 진행할 수 있습니다.
