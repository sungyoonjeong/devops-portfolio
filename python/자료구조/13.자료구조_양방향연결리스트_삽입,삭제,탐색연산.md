# 📑 강의 자료: 양방향 연결 리스트의 연산과 성능 분석

양방향 연결 리스트(Doubly Linked List)는 모든 연산의 기초가 되는 `splice`를 활용하여 삽입, 삭제, 이동 연산을 매우 직관적이고 효율적으로 처리합니다.

## 1. 이동(Move) 연산
특정 노드를 리스트 내의 다른 위치로 옮기는 연산입니다. 한방향 연결 리스트에는 없는 개념이지만, 양방향에서는 `splice`를 이용해 한 줄로 구현 가능합니다.

*   **`move_after(a, x)`**: 노드 `a`를 노드 `x` 다음으로 이동시킵니다. 내부적으로 `splice(a, a, x)`를 호출합니다.
*   **`move_before(a, x)`**: 노드 `a`를 노드 `x` 전으로 이동시킵니다. `splice(a, a, x.prev)`를 호출하여 구현합니다.

## 2. 삽입(Insertion) 연산
새로운 데이터를 리스트의 특정 위치에 추가합니다.

*   **기본 원리**: `Node(key)`로 새로운 노드 객체를 생성한 후, 이를 `move_after`나 `move_before`를 통해 목적 위치에 배치합니다.
*   **주요 함수**:
    *   **`push_front(key)`**: 리스트의 맨 앞(더미 헤드 바로 뒤)에 삽입합니다. `insert_after(self.head, key)`와 같습니다.
    *   **`push_back(key)`**: 리스트의 맨 뒤(더미 헤드 바로 전)에 삽입합니다. `insert_before(self.head, key)`와 같습니다.

## 3. 삭제(Deletion) 연산
특정 노드를 리스트에서 제거하고 앞뒤 노드를 직접 연결합니다.

*   **`remove(x)`**: 노드 `x`를 삭제합니다.
    *   **로직**: `x`의 이전 노드(`x.prev`)의 `next`를 `x.next`에 연결하고, `x`의 다음 노드(`x.next`)의 `prev`를 `x.prev`에 연결합니다.
    *   **예외 처리**: `x`가 `None`이거나 더미 노드(`head`)인 경우에는 삭제 연산을 수행하지 않습니다.
*   **`pop_front()` / `pop_back()`**: 각각 리스트의 첫 번째 노드나 마지막 노드를 `remove` 함수에 전달하여 삭제합니다.

## 4. 탐색(Search) 연산
특정 키 값을 가진 노드를 찾는 연산입니다.

*   **작동 방식**: 더미 헤드 다음 노드부터 시작하여 리스트를 한 바퀴 돌아 **다시 더미 헤드로 돌아올 때까지** 차례대로 비교합니다.
*   **결과**: 키를 찾으면 해당 노드를 반환하고, 한 바퀴를 다 돌았는데도 없으면 `None`을 반환합니다.
*   더미 헤드가 왜 필요한가?
→ 실제 데이터 없이 리스트의 시작점 역할
→ 덕분에 첫 번째·마지막 노드 삽입/삭제 시
   예외 처리 없이 동일한 코드로 처리 가능
→ head.next = head.prev = head (빈 리스트 초기 상태)

# splice(a, b, x): 노드 a~b 구간을 떼어내어 x 다음에 붙임
def splice(self, a, b, x):
    # 1. a~b 구간을 원래 위치에서 분리
    ap = a.prev
    bn = b.next
    ap.next = bn
    bn.prev = ap

    # 2. a~b 구간을 x 다음에 연결
    xn = x.next
    x.next = a
    a.prev = x
    b.next = xn
    xn.prev = b

## 5. 연산별 시간 복잡도 분석
양방향 연결 리스트의 가장 큰 장점은 **삭제와 삽입이 매우 빠르다**는 것입니다.

| 연산 유형 | 주요 연산 | 시간 복잡도 | 이유 |
| :--- | :--- | :--- | :--- |
| **탐색** | `search(key)` | **$O(n)$** | 최악의 경우 모든 노드를 다 확인해야 함 |
| **이동** | `move_after/before` | **$O(1)$** | `splice`를 이용한 6개의 링크 수정 |
| **삽입** | `push_front/back` | **$O(1)$** | 위치가 고정되어 있어 즉시 삽입 가능 |
| **삭제** | `remove(x)` | **$O(1)$** | **노드 `x`를 알고 있다면** 즉시 링크 수정 가능 |

> **⚠️ 주의 사항**: 특정 키 값을 가진 노드를 삭제하려면 `search`($O(n)$)를 먼저 수행하여 노드를 찾은 뒤 `remove`($O(1)$)를 호출해야 하므로, 전체 과정은 $O(n)$이 소요될 수 있습니다.

## 6. 자료구조 비교 요약
*   **배열(Array)**: 맨 앞 삽입/삭제 시 데이터를 밀어야 하므로 $O(n)$이 걸립니다.
*   **한방향 연결 리스트**: 마지막 노드 삭제(`pop_back`) 시 이전 노드를 찾기 위해 헤드부터 가야 하므로 $O(n)$이 걸립니다.
*   **양방향 연결 리스트**: 앞뒤 링크가 모두 있어 **모든 위치에서의 삽입/삭제가 $O(1)$**로 매우 효율적입니다.
*   **양방향 연결 리스트가 실제로 쓰이는 곳**:
→ LRU 캐시 (Least Recently Used)
   → K8s Pod 캐시 정책에도 사용
→ 브라우저 방문 기록 (앞으로/뒤로 가기)
→ 텍스트 에디터의 실행 취소(Undo) 기능
