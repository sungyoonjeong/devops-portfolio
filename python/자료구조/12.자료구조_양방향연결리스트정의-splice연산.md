# 📑 강의 자료: 양방향 연결 리스트(Doubly Linked List)와 핵심 연산

## 1. 한방향 연결 리스트의 한계와 양방향의 필요성
*   **한계**: 한방향 연결 리스트는 `next` 링크만 있어 특정 노드에서 이전 노드로 되돌아갈 수 없습니다.
*   **문제 상황**: 테일(Tail) 노드를 삭제하려면 그 직전 노드의 링크를 수정해야 하는데, 이를 찾기 위해 헤드부터 전체 리스트를 탐색해야 하므로 $O(n)$ 시간이 걸립니다.
*   **해결책**: 각 노드에 오른쪽 링크(`next`)와 함께 **왼쪽 링크(`prev`)**를 추가하여 상수의 시간($O(1)$) 내에 이전 노드로 접근할 수 있게 합니다.

## 2. 원형 양방향 연결 리스트 (Circular Doubly Linked List)
단순한 선형 구조가 아니라, 양 끝이 연결된 **원형 구조**를 채택하면 삽입과 삭제가 훨씬 간결해집니다.
*   **더미 노드(Dummy Node)**: 실제 데이터를 담지 않지만 리스트의 시작과 끝을 연결하는 '마커' 역할을 하는 특별한 노드입니다.
*   **원형 구조**: 테일의 `next`가 헤드(더미)를 가리키고, 헤드의 `prev`가 테일을 가리킵니다.
*   **장점**: 리스트가 비어 있거나 노드가 하나일 때의 예외 처리가 줄어들어 코드가 우아해집니다.

## 3. 클래스 구조 및 초기화
*   **노드(Node) 클래스**: `key`, `next`, `prev` 세 가지 멤버를 가집니다.
*   **초기 상태**: 비어 있는 리스트에서 더미 노드는 `next`와 `prev`가 모두 **자기 자신**을 가리키도록 설정합니다.
*   **리스트 관리**: `head`(더미 노드 지칭)와 현재 노드의 개수를 나타내는 `size` 정보를 유지합니다.

## 4. 핵심 연산: 스플라이스(splice)
`splice(a, b, x)`는 노드 `a`부터 `b`까지의 구간을 잘라내어 노드 `x` 뒤에 붙이는 연산으로, 모든 삽입과 이동의 기초가 됩니다.

### 4.1 조건 (Constraints)
1.  노드 `a`를 계속 따라가면 노드 `b`가 나와야 합니다.
2.  잘라낼 구간(`a`~`b`) 사이에 **더미(헤드) 노드**가 있어서는 안 됩니다.
3.  잘라낼 구간 사이에 이동 목적지인 **노드 `x`**가 포함되어서는 안 됩니다.

### 4.2 작동 원리 (6개의 링크 수정)
이 연산은 총 6개의 연결 관계를 수정하여 완성됩니다.
1.  **Cut (2개 수정)**: 원래 리스트에서 `a`와 `b` 구간을 떼어냅니다. `a`의 이전 노드와 `b`의 다음 노드를 서로 연결합니다.
2.  **Paste (4개 수정)**: 잘라낸 구간을 `x`와 그 다음 노드(`xn`) 사이에 끼워 넣습니다.
    *   `x`의 `next` → `a` 연결 / `a`의 `prev` → `x` 연결.
    *   `b`의 `next` → `xn` 연결 / `xn`의 `prev` → `b` 연결.

## 5. 결론 및 요약
*   양방향 연결 리스트는 관리할 링크가 두 배로 늘어나지만, **삭제와 삽입 연산을 $O(1)$ 시간**에 처리할 수 있다는 강력한 장점이 있습니다.
*   **`splice` 연산** 하나만 제대로 구현하면, 이를 응용하여 노드의 삽입, 삭제, 이동 등 다양한 연산을 매우 효율적으로 수행할 수 있습니다.

### 1. Node 클래스
각 노드는 데이터인 `key`와 양옆을 가리키는 `next`, `prev` 링크로 구성됩니다.

```python
class Node:
    def __init__(self, key=None):
        self.key = key          # 노드에 저장할 데이터(키값)를 설정하며, 기본값은 None입니다.
        self.next = self        # 다음 노드를 가리키는 링크로, 초기화 시 자기 자신을 가리킵니다.
        self.prev = self        # 이전 노드를 가리키는 링크로, 초기화 시 자기 자신을 가리킵니다.
```

### 2. DoublyLinkedList 클래스
이 클래스는 **더미 노드(Dummy Node)**를 헤드로 가지며, 리스트의 크기 정보를 관리합니다.

```python
class DoublyLinkedList:
    def __init__(self):
        # 더미 노드를 하나 생성하여 헤드가 이를 가리키도록 합니다. 
        # 이 더미 노드는 리스트의 시작을 알리는 마커 역할을 합니다.
        self.head = Node()      
        self.size = 0           # 현재 리스트에 저장된 실제 데이터 노드의 개수를 0으로 초기화합니다.
```

### 💡 주요 특징 설명
*   **원형 구조**: `Node` 클래스에서 `next`와 `prev`를 `self`(자기 자신)로 초기화하는 이유는 비어 있는 리스트 상태에서도 **원형 연결 구조**를 유지하기 위함입니다.,
*   **더미 노드 활용**: `DoublyLinkedList`의 `head`는 항상 데이터가 없는 더미 노드를 가리키며, 이를 통해 삽입과 삭제 연산을 더 일관되고 우아하게 처리할 수 있습니다.,
*   **상수 시간 연산**: 이렇게 양방향으로 연결된 구조를 사용하면, 한방향 리스트와 달리 이전 노드를 즉시 알 수 있어 삭제 연산 등을 **상수 시간($O(1)$)** 내에 처리할 수 있는 장점이 있습니다.

**스플라이스(splice)** 연산은 양방향 연결 리스트에서 특정 구간의 노드들을 떼어내어 다른 위치에 붙이는 가장 핵심적이고 기초적인 연산입니다. 이 연산을 통해 노드의 이동, 삽입, 삭제를 매우 효율적으로 처리할 수 있습니다.

### 1. 스플라이스의 정의와 조건
`splice(a, b, x)`는 **노드 `a`부터 `b`까지의 구간을 잘라내어(Cut), 노드 `x` 바로 뒤에 붙여넣는(Paste)** 작업입니다.

*   **입력값**: 세 개의 **노드 객체** `a`, `b`, `x` (키값이 아님).
*   **제약 조건**:
    1.  `a`를 계속 따라가면 언젠가 `b`가 나와야 합니다 (즉, `a`가 `b`보다 앞서거나 같아야 함).
    2.  잘라낼 구간(`a`~`b`) 사이에 **헤드(더미) 노드**가 포함되어서는 안 됩니다.
    3.  잘라낼 구간(`a`~`b`) 사이에 이동 목적지인 **노드 `x`**가 포함되어 있어서는 안 됩니다.

---

### 2. 스플라이스 과정 (그림 설명)

#### **[단계 1: 연산 전 상태]**
원래 리스트에서 `a`~`b` 구간과 목적지 `x`의 모습입니다.
*   `ap`: `a`의 이전 노드 (a.prev)
*   `bn`: `b`의 다음 노드 (b.next)
*   `xn`: `x`의 다음 노드 (x.next)

```text
리스트 A:  ...[ap] <-> [a] <-> (노드들) <-> [b] <-> [bn]...
리스트 B:  ...[x] <-> [xn]...
```

#### **[단계 2: Cut - 구간 떼어내기]**
`a`와 `b` 사이의 구간을 기존 리스트에서 분리하고, 남겨진 양쪽(`ap`와 `bn`)을 서로 연결합니다.
*   **수정된 링크**: `ap.next = bn`, `bn.prev = ap`

```text
기존 리스트: ...[ap] <----------------------> [bn]...  (연결 완료)
잘라낸 구간:           [a] <-> (노드들) <-> [b]         (공중에 뜸)
```

#### **[단계 3: Paste - 목적지에 붙이기]**
떼어낸 구간(`a`~`b`)을 `x`와 `xn` 사이에 끼워 넣습니다.
*   **수정된 링크**: `x`와 `a` 연결, `b`와 `xn` 연결 (총 4개)

```text
최종 리스트: ...[x] <-> [a] <-> (노드들) <-> [b] <-> [xn]...
```

---

### 3. 코드 로직 (6개의 링크 수정)
이 연산은 총 **6개의 링크**를 수정함으로써 상수 시간 내에 완료됩니다.

```python
# 1. 주변 노드 정의
ap = a.prev
bn = b.next
xn = x.next

# 2. Cut: 원래 리스트에서 a~b 구간 제외 (2개 링크)
ap.next = bn      # a 이전 노드의 다음을 b 이후 노드로 연결
bn.prev = ap      # b 이후 노드의 이전을 a 이전 노드로 연결

# 3. Paste: x 뒤에 a~b 구간 삽입 (4개 링크)
x.next = a        # x의 다음을 a로 연결
a.prev = x        # a의 이전을 x로 연결
b.next = xn       # b의 다음을 원래 x의 다음 노드로 연결
xn.prev = b       # 원래 x의 다음 노드의 이전을 b로 연결
```

### 4. 왜 중요한가?
이 `splice` 연산은 양방향 연결 리스트의 거의 모든 기능을 수행하는 **기본 단위**가 됩니다. 예를 들어:
*   **이동**: `splice(a, a, x)`를 호출하면 노드 `a` 하나만 `x` 뒤로 옮길 수 있습니다.
*   **삽입**: 새로운 노드를 만든 후, 그 노드를 `a`, `b` 자리에 넣고 `splice`하면 특정 위치에 삽입이 됩니다.
*   **삭제**: 특정 구간을 잘라낸 후 어디에도 붙이지 않으면(또는 더미 노드에 붙이면) 삭제 연산이 됩니다.
