# 🌲 [강의 자료] 이진 트리(Binary Tree)의 정의와 순회

이 자료는 이진 트리의 기본적인 구조와 이를 프로그래밍 언어로 구현하는 방법, 그리고 트리의 모든 노드를 방문하는 세 가지 순회 방식에 대해 다룹니다.

## 1. 이진 트리(Binary Tree)의 개요

### 1.1 정의
*   **이진 트리**는 각 노드의 **자식 노드가 없거나, 하나이거나, 최대 2개까지** 있을 수 있는 트리 구조를 말합니다.
*   일반적인 트리(자식이 여러 개인 트리)에 비해 삽입과 삭제 연산이 덜 복잡하기 때문에 자료구조에서 가장 많이 활용되는 형태입니다.

### 1.2 표현법 (Representation)
1.  **배열 또는 리스트 표현**: 대표적으로 '힙(Heap)'이 이 방식을 사용하지만, 트리의 모양에 따라 메모리 낭비가 발생할 수 있습니다.
2.  **노드와 링크 직접 표현**: 객체 지향 프로그래밍에서 `Node` 클래스를 선언하고 링크를 통해 부모-자식 관계를 직접 연결하는 방식입니다.

---

## 2. 노드(Node) 클래스 구현

이진 트리의 각 노드는 데이터와 함께 주변 노드와의 관계를 나타내는 링크 정보를 가집니다.

### 2.1 멤버 변수 (속성)
*   **Key**: 노드에 저장되는 실제 값 (필요시 Value도 포함 가능).
*   **Left**: 왼쪽 자식 노드를 가리키는 링크.
*   **Right**: 오른쪽 자식 노드를 가리키는 링크.
*   **Parent**: 부모 노드를 가리키는 링크. (**루트 노드**는 부모가 없으므로 이 값이 `None`이 됩니다.)

### 2.2 파이썬 코드 구현
```python
class Node:
    def __init__(self, key):
        self.key = key         # 키값 설정
        self.parent = None     # 부모 노드 링크 (기본값 None)
        self.left = None       # 왼쪽 자식 링크 (기본값 None)
        self.right = None      # 오른쪽 자식 링크 (기본값 None)

    def __str__(self):
        # 노드를 출력할 때 키값을 문자열로 반환하여 확인 가능하게 함
        return str(self.key)
```

## 3. 이진 트리의 순회 (Traversal)

**순회**란 트리에 있는 모든 노드의 키값들을 빠짐없이 차례대로 방문하여 출력하거나 처리하는 과정을 말합니다. 방문 순서에 따라 크게 세 가지 방법이 있습니다.

### 3.1 세 가지 순회 방식
방문 기준은 **자기 자신(M)**, **왼쪽 서브트리(L)**, **오른쪽 서브트리(R)**입니다. 모든 방식에서 **L은 항상 R보다 먼저** 방문합니다.

| 종류 | 방문 순서 | 특징 |
| :--- | :--- | :--- |
| **전위 순회 (Pre-order)** | **M** → L → R | **자기 자신**을 가장 먼저 방문하고 자식들로 내려감 |
| **중위 순회 (In-order)** | L → **M** → R | 왼쪽을 모두 본 후 **자기 자신**을 방문하고 오른쪽으로 감 |
| **후위 순회 (Post-order)** | L → R → **M** | 자식들을 모두 방문한 후 마지막에 **자기 자신**을 방문 |

### 3.2 재귀적 코드 구현 예시
```python
def preorder(self):
    if self is not None:
        print(self.key)         # M (방문)
        if self.left:
            self.left.preorder() # L (왼쪽으로 재귀 호출)
        if self.right:
            self.right.preorder() # R (오른쪽으로 재귀 호출)
```
*   **중위 순회**와 **후위 순회**는 위 코드에서 `print(self.key)`의 위치를 각각 중간(`L`과 `R` 사이)이나 마지막(`R` 뒤)으로 옮기면 구현할 수 있습니다.

---

## 4. 트리 복원 (Reconstruction)

*   트리의 실제 모양을 모르더라도 **전위 순회(Pre-order) 결과**와 **중위 순회(In-order) 결과**가 주어지면, 이 두 정보를 조합하여 원래 트리의 구조를 완벽하게 추측하고 복원할 수 있습니다.
*   이는 각 순회 방식이 가진 고유한 노드 방문 순서의 논리를 거꾸로 추적함으로써 가능합니다.
