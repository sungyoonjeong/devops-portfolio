# [강의 자료] 팀 정렬 (Tim Sort): 현대 프로그래밍의 표준 정렬

이 자료는 파이썬(Python)을 비롯해 자바(Java), 안드로이드 등 수많은 시스템에서 채택하고 있는 가장 실용적인 정렬 알고리즘인 **팀 정렬(Tim Sort)**의 동작 원리를 다룹니다.

## 1. 팀 정렬이란 무엇인가?

*   **정의:** 2002년 팀 피터스(Tim Peters)가 고안한 알고리즘으로, **삽입 정렬(Insertion Sort)**과 **병합 정렬(Merge Sort)**의 장점을 결합한 하이브리드 정렬입니다.
*   **사용처:** 파이썬의 `sort()` 함수, 자바의 객체 정렬, 스위프트, 안드로이드 등 현대 거의 모든 언어의 기본 정렬로 사용됩니다.
*   **성능 요약:**
    *   **최악의 시간 복잡도:** $O(n \log n)$.
    *   **최선의 시간 복잡도:** $O(n)$ (이미 정렬된 데이터의 경우).
    *   **공간 복잡도:** 추가 메모리 $O(n)$ 사용 (In-place 아님).
    *   **안정성(Stable):** 동일한 값의 상대적 순서가 유지됩니다.

---

## 2. 왜 팀 정렬을 쓰는가? (실생활 데이터의 특성)

실제 데이터는 완전히 무작위(Random)이기보다는 **이미 어느 정도 정렬된 부분**이 섞여 있는 경우가 많습니다. 팀 정렬은 이러한 실생활 데이터의 경향성을 파악하여 더 빠르게 정렬하도록 설계되었습니다.

---

## 3. 핵심 동작 원리: 런(Run)과 병합(Merge)

팀 정렬은 전체 데이터를 한꺼번에 정렬하지 않고, 작은 단위인 **'런(Run)'**으로 쪼개어 처리합니다.

### [Step 1] 런(Run) 만들기
1.  **분할:** 데이터를 앞에서부터 읽으며 증가하거나 감소하는 연속적인 구간을 찾습니다.
2.  **최소 크기 유지:** 런의 최소 크기는 보통 **32~64개**입니다. 만약 찾은 런이 너무 작으면 **삽입 정렬**을 사용하여 부족한 만큼 데이터를 채워 넣습니다.
    *   **왜 삽입 정렬인가?** 데이터 개수가 적을 때는 퀵 정렬이나 병합 정렬보다 삽입 정렬이 훨씬 빠르고 효율적이기 때문입니다.
3.  **내림차순 처리:** 내림차순으로 정렬된 런은 단순히 뒤집어서(Reverse) 오름차순으로 만듭니다.

### [Step 2] 스택(Stack)을 이용한 지능적 병합
런들이 만들어지면 무작정 합치는 것이 아니라 **스택**에 넣고 관리합니다.
*   **병합 조건:** 스택의 가장 위에 있는 런 3개의 크기를 비교하여, 크기가 비슷한 것끼리 먼저 병합합니다 (피보나치 수와 유사한 방식).
*   **이유:** 크기가 비슷한 런끼리 합쳐야 병합 과정이 효율적이며, 전체 스택의 크기를 작게($O(\log n)$) 유지할 수 있습니다.

---

## 4. 팀 정렬만의 '미친' 최적화 기법

### ① 메모리 아끼기 (Memory Optimization)
일반적인 병합 정렬은 전체 데이터 크기만큼의 임시 배열($n$)이 필요합니다. 팀 정렬은 병합할 두 런 중 **더 작은 쪽의 크기만큼만** 임시 공간을 만들어 메모리 사용량을 대폭 줄입니다.

### ② 갤로핑 모드 (Galloping Mode: 질주 모드)
병합할 때 한쪽 런의 데이터가 연속적으로 선택되는 경우가 있습니다. 이때는 하나씩 비교하지 않고 **2, 4, 8, 16...**과 같이 기하급수적으로 건너뛰며 비교하여 병합 위치를 빠르게 찾습니다. 이것을 '질주(Galloping) 모드'라고 부릅니다.

### ③ 이진 탐색 삽입 정렬 (Binary Insertion Sort)
런 내부에서 삽입 정렬을 수행할 때, 들어갈 위치를 찾기 위해 앞에서부터 하나씩 보는 게 아니라 **이진 탐색(Binary Search)**을 사용하여 비교 횟수를 줄입니다.

---

## 5. 요약: 팀 정렬의 강점

1.  **실제 데이터에 강함:** 이미 정렬된 구간을 최대한 활용하여 $O(n)$까지 속도를 높입니다.
2.  **안전하고 빠름:** $O(n \log n)$을 보장하면서도 스테이블(Stable)한 정렬입니다.
3.  **지능적 메모리 사용:** 임시 공간을 최소화하고 스택을 통해 병합 효율을 극대화합니다.
