# 🌲 [강의 자료] 이진탐색트리(Binary Search Tree)의 정의와 핵심 연산

이진탐색트리(BST)는 이진 트리의 한 종류로, 데이터를 효율적으로 조직화하여 **탐색, 삽입, 삭제** 연산을 빠르게 수행할 수 있도록 설계된 자료구조입니다.

## 1. 이진탐색트리의 정의와 성질

### 1.1 BST의 핵심 조건
모든 노드에 대해 다음 규칙을 만족해야 합니다.
*   **왼쪽 서브트리의 키값**: 부모 노드의 키값보다 **작거나 같아야** 합니다.
*   **오른쪽 서브트리의 키값**: 부모 노드의 키값보다 **커야** 합니다.
*   이 규칙은 루트 노드뿐만 아니라 트리의 **모든 노드에 재귀적으로 적용**됩니다.

### 1.2 클래스 구조
*   **Node 클래스**: `key`, `left`, `right`, `parent` 링크를 포함합니다.
*   **BST 클래스**: 트리의 시작점인 `root` 노드와 전체 노드 개수를 관리하는 `size` 정보를 가집니다.

## 2. 탐색 연산: `find_location(key)`

모든 연산의 기초가 되는 함수로, 특정 키값의 위치를 찾거나 삽입될 위치를 결정합니다.

*   **작동 원리**: 루트부터 시작해 찾고자 하는 키와 현재 노드를 비교합니다. 키가 작으면 왼쪽, 크면 오른쪽으로 이동합니다.
*   **반환 값**:
    1.  키를 **찾은 경우**: 해당 노드를 반환합니다.
    2.  키를 **찾지 못한 경우**: 해당 키가 삽입될 위치의 **부모 노드**를 반환합니다.
*   **시간 복잡도**: 트리의 높이($h$)에 비례하는 **$O(h)$**가 소요됩니다.

```python
def find_location(self, key):
    if self.size == 0: return None # 트리가 비어있는 경우
    
    p = None # 부모 노드를 추적 
    v = self.root # 루트부터 탐색 시작 
    
    while v is not None: # 노드가 존재할 때까지 반복 
        if v.key == key: return v # 키를 찾으면 해당 노드 반환 
        p = v # 이동 전 현재 노드를 부모로 설정 
        if v.key < key:
            v = v.right # 키가 더 크면 오른쪽으로 이동 
        else:
            v = v.left # 키가 더 작으면 왼쪽으로 이동 
            
    return p # 키를 못 찾으면 삽입될 위치의 부모 노드 반환 
```

## 3. 삽입 연산: `insert(key)`

새로운 데이터를 힙 성질을 유지하며 적절한 위치에 추가합니다.

1.  **위치 탐색**: `find_location`을 호출하여 부모가 될 노드(`p`)를 찾습니다.
2.  **노드 생성**: 새로운 노드(`v`)를 생성합니다.
3.  **링크 연결**:
    *   트리가 비어있으면 `v`를 루트로 설정합니다.
    *   비어있지 않다면, 부모 `p`의 키와 비교하여 `v`를 `p`의 왼쪽 또는 오른쪽 자식으로 연결합니다.
    *   만약 이미 존재하는 키라면 삽입하지 않고 중복 메시지를 출력합니다.

```python
def insert(self, key):
    p = self.find_location(key) # 삽입될 위치의 부모 노드 탐색 
    
    # 1. 트리가 비어있거나 중복된 키가 없는 경우에만 삽입 수행 
    if p is None or p.key != key:
        v = Node(key) # 새로운 노드 생성 
        if p is None:
            self.root = v # 트리가 비어있었다면 루트로 설정 
        else:
            v.parent = p # 새 노드의 부모 설정 
            if p.key > key:
                p.left = v # 부모보다 작으면 왼쪽 자식 
            else:
                p.right = v # 부모보다 크면 오른쪽 자식 
        self.size += 1 # 크기 증가 
        return v # 삽입된 노드 반환 
    else:
        print("Key already exists") # 중복 키 처리 
        return None
```

## 4. 삭제 연산: 두 가지 전략

노드 `x`를 삭제할 때는 트리의 연결이 끊어지지 않도록 재구성하는 것이 중요합니다.

### 4.1 딜리트 바이 머징 (Delete by Merging)
삭제할 노드 `x`의 자리를 왼쪽 서브트리($L$)가 대체하도록 하고, 오른쪽 서브트리($R$)를 $L$의 적절한 위치에 붙이는 방식입니다.
*   **절차**:
    1.  $L$에서 가장 큰 키값을 가진 노드($m$)를 찾습니다 ($L$에서 계속 오른쪽으로 이동).
    2.  $m$의 오른쪽 자식으로 $R$을 연결합니다 (이유: $R$의 모든 값은 $L$의 모든 값보다 크기 때문).
    3.  $x$의 부모 노드와 $L$을 연결하여 $x$를 완전히 제거합니다.
*   **특징**: 노드의 이동이 발생하며 트리 구조가 크게 변할 수 있습니다.

```python
def delete_by_merging(self, x):
    a = x.left  # 왼쪽 자식 
    b = x.right # 오른쪽 자식 
    pt = x.parent # 삭제할 노드의 부모 
    
    # 1. 삭제 후 그 자리를 대체할 노드(c)와 가장 큰 노드(m) 결정
    if a is not None:
        c = a # 왼쪽 서브트리가 통째로 올라옴 
        m = a
        while m.right is not None: # 왼쪽에서 가장 큰 노드 탐색
            m = m.right
        
        # 2. 가장 큰 노드 m의 오른쪽에 원래 오른쪽 서브트리 b 연결 
        m.right = b
        if b is not None:
            b.parent = m # 부모 링크 업데이트 
    else:
        c = b # 왼쪽 자식이 없으면 오른쪽이 바로 올라옴 

    # 3. 삭제될 노드 x의 부모(pt)와 대체 노드(c) 연결 
    if pt is None: # x가 루트였던 경우 
        self.root = c
        if c is not None: c.parent = None
    else:
        if c is not None: c.parent = pt # 대체 노드의 부모 설정
        if pt.key < c.key: # 부모의 왼쪽 자식이었다면 
            pt.right = c
        else: # 부모의 오른쪽 자식이었다면 
            pt.left = c
            
    self.size -= 1 # 크기 감소 
```

### 4.2 딜리트 바이 카핑 (Delete by Copying)
노드 자체를 삭제하는 대신, 다른 노드의 키값을 복사해와서 덮어쓰는 방식입니다.
*   **절차**:
    1.  $x$의 왼쪽 서브트리에서 가장 큰 노드($m$)를 찾습니다.
    2.  $m$의 키값을 $x$의 위치로 **복사**합니다. 이제 $x$는 논리적으로 삭제된 효과를 냅니다.
    3.  실제로는 키가 빠져나간 노드 $m$을 삭제하고, $m$의 자식 노드들을 $m$의 부모와 연결합니다.
*   **특징**: $x$ 노드 객체 자체가 유지되므로 링크 수정이 상대적으로 적습니다.

```python
def delete_by_copying(self, x):
    # 1. 왼쪽 서브트리(L)에서 가장 큰 키값을 가진 노드(m)를 찾음 (Predecessor)
    # x의 왼쪽 자식에서 시작하여 계속 오른쪽 자식을 따라 내려감
    m = x.left
    while m.right is not None:
        m = m.right [1, 2]

    # 2. 찾은 노드 m의 키값을 삭제할 노드 x의 위치로 복사함
    # 이제 x 노드는 논리적으로 트리에서 사라지고 m의 값으로 대체됨
    x.key = m.key [1, 3]

    # 3. 키를 빌려준 노드 m은 이제 필요 없으므로 물리적으로 제거함
    # m은 서브트리에서 가장 큰 값이므로 '오른쪽 자식'이 절대 존재할 수 없음 
    pt_m = m.parent     # m의 부모 노드
    c_m = m.left        # m의 왼쪽 자식 노드 (있을 수도, 없을 수도 있음)

    # m의 부모와 m의 유일한 자식인 c_m을 직접 연결하여 m을 우회시킴 
    if pt_m.left == m:
        pt_m.left = c_m
    else:
        pt_m.right = c_m

    # 자식 노드가 존재한다면 부모 링크도 업데이트함
    if c_m is not None:
        c_m.parent = pt_m [3, 4]

    self.size -= 1 # 트리 전체 노드 개수 감소 
```
## 단계별 상세 로직 설명
*   **대체 노드(m) 찾기**: 삭제할 노드 x의 왼쪽 서브트리(L)에 있는 모든 값은 x보다 작고, 오른쪽 서브트리(R)에 있는 모든 값은 x보다 큽니다
*   따라서 L에서 가장 큰 값인 m을 뽑아 x 자리에 놓으면, 여전히 왼쪽보다는 크고 오른쪽보다는 작은 이진탐색트리의 규칙을 완벽하게 유지할 수 있습니다

*   **값의 복사**: 노드 객체 사이의 복잡한 링크를 모두 끊고 다시 연결하는 대신, x.key = m.key와 같이 값만 전달하기 때문에 구현 논리가 상대적으로 명확합니다

*   물리적 삭제: 값을 빌려준 노드 m은 반드시 삭제되어야 합니다. 이때 m은 오른쪽 자식이 없는 상태이므로, 단순히 자신의 부모와 자신의 왼쪽 자식을 연결해주는 것만으로 간단히 삭제가 완료됩니다
.

## 5. 성능 분석 및 결론

### 5.1 시간 복잡도
*   **탐색, 삽입, 삭제**: 모두 트리의 높이에 비례하는 **$O(h)$** 시간이 걸립니다.
*   **최악의 경우**: 데이터가 정렬된 순서대로 들어와 트리가 한쪽으로 치우치면(Skewed), 높이 $h$가 노드 개수 $n$과 같아져 **$O(n)$**으로 느려집니다.

### 5.2 균형 이진 탐색 트리(Balanced BST)의 필요성
*   연산 효율을 극대화하기 위해서는 트리의 높이 $h$를 최소화($\log n$에 가깝게) 유지해야 합니다.
*   이를 위해 스스로 균형을 잡는 **Balanced BST**가 다음 단계의 핵심 주제가 됩니다.
