# 📑 강의 자료: 해시 테이블 - 오픈 어드레싱과 선형 조사법

## 1. 오픈 어드레싱 (Open Addressing) 개요
해시 함수가 완벽하지 않은 이상 **충돌(Collision)**은 불가피합니다. 오픈 어드레싱은 충돌이 발생했을 때, 테이블 내의 다른 빈 슬롯을 찾아 데이터를 저장하는 방식입니다.

*   **핵심 원리**: 현재 슬롯이 차 있다면, 정해진 규칙에 따라 주위의 빈칸을 '찔러보며(Probing)' 빈 공간을 찾습니다.
*   **주요 조사 방법**:
    *   **선형 조사법 (Linear Probing)**: 바로 다음 슬롯을 순차적으로 확인.
    *   **이차 조사법 (Quadratic Probing)**: 보폭을 제곱으로 늘리며 확인.
    *   **이중 해싱 (Double Hashing)**: 별도의 해시 함수를 하나 더 사용해 보폭을 결정.

---

## 2. 선형 조사법 (Linear Probing)의 작동 원리

### 2.1 삽입 및 탐색 규칙
*   충돌 발생 시 **한 칸씩 밑으로($i+1, i+2, \dots$)** 내려가며 빈칸을 찾습니다.
*   테이블의 끝(마지막 인덱스)에 도달하면 다시 **0번 슬롯으로 돌아가는 원형(Circular) 구조**를 가집니다.
*   **탐색** 시에도 삽입할 때와 같은 경로를 따라가며, 찾는 값이 없더라도 **빈 슬롯을 만날 때까지** 계속 확인해야 합니다. 빈 슬롯을 만났다는 것은 해당 데이터가 애초에 저장되지 않았음을 의미하기 때문입니다.

### 2.2 클러스터링(Clustering) 문제
*   **군집 현상**: 키 값들이 특정 영역의 슬롯에 연속적으로 모이는 현상을 '클러스터'라고 합니다.
*   **성능 저하**: 클러스터가 커지면 빈칸을 찾기 위해 더 많은 슬롯을 뒤져야 하므로 삽입과 탐색 시간이 $O(n)$에 가까워지며 효율이 급격히 떨어집니다.

---

## 3. 핵심 연산 구현 (Pseudo-code)

### 3.1 `find_slot(key)`: 슬롯 위치 찾기
모든 연산의 기초가 되는 함수로, 키가 있는 위치나 삽입될 위치를 반환합니다.

```python
def find_slot(self, key):
    i = self.hash_function(key) # 초기 해시 위치 계산
    start = i
    while self.table[i] is occupied: # 슬롯이 차 있는 동안 반복
        if self.table[i].key == key: # 이미 키가 존재하면 해당 위치 반환
            return i
        i = (i + 1) % self.m # 다음 칸 확인 (원형 구조)
        if i == start: # 한 바퀴 다 돌았는데도 자리가 없으면
            return "Full" # 테이블이 가득 참
    return i # 빈 슬롯 위치 반환 (키가 없는 경우)
```

### 3.2 `set(key, value)`: 삽입 및 업데이트
1.  `find_slot`으로 위치를 찾습니다.
2.  결과가 `Full`이면 테이블 크기를 키워야 합니다.
3.  해당 위치에 이미 키가 있으면 값을 업데이트하고, 비어 있으면 새로 삽입합니다.

## 4. 삭제(Remove) 연산: 가장 까다로운 부분
단순히 특정 노드를 지우고 빈칸으로 남겨두면 **탐색의 고리**가 끊어지는 문제가 발생합니다.

*   **문제**: A 때문에 밀려난 B가 있을 때, 중간의 A를 그냥 지우면 나중에 B를 찾을 때 빈칸을 만나 탐색이 중단됩니다.
*   **해결 방법**: 지워진 빈칸을 메우기 위해 아래에 있는 노드들을 **재배치**해야 합니다.
*   **이동 조건**: 아래에 있는 노드 $j$의 원래 해시 위치 $k$가 빈칸 $i$와 현재 위치 $j$ 사이에 있다면, $j$를 $i$로 옮겨 고리를 이어줍니다.

# [자료구조] 선형 조사법(Linear Probing)의 데이터 삭제 알고리즘

이 알고리즘은 해시 테이블에서 **선형 조사법(Linear Probing)**을 사용할 때, 특정 키(Key)를 안전하게 삭제(`remove`)하는 과정을 구현한 의사코드(Pseudocode)입니다.

## 1. 개요 및 핵심 개념
선형 조사법에서는 중간의 데이터를 단순히 빈 슬롯(`None`)으로 지워버리면, 해시 충돌로 인해 뒤쪽에 저장된 데이터를 찾지 못하는 **검색 단절 문제**가 발생합니다. 
따라서 본 알고리즘은 **지워진 빈 자리를 메울 수 있는 적절한 뒤쪽 데이터를 찾아서 앞으로 당겨오는(이동시키는) 과정**을 반복하여 해시 테이블의 연속성을 유지합니다.

---

## 2. 알고리즘 의사코드 (Pseudocode)

```python
def remove(key):
    # 1. 해시 함수를 통해 key가 저장되어 있어야 할 인덱스를 찾음
    i = find_slot(key)
    
    # 2. 해당 슬롯이 처음부터 비어있다면 삭제할 데이터가 없음
    if H[i] is unoccupied: 
        return None
    
    # 3. 인덱스 초기화 (i: 빈 슬롯 위치, j: 이동할 후보 슬롯 위치)
    j = i 
    
    while True:
        H[i] = None  # 현재 i 위치의 데이터를 삭제하여 빈 슬롯으로 만듦
        
        while True:  # 이사해야 할 적절한 H[j]를 찾는 루프
            j = (j + 1) % m  # 다음 슬롯으로 이동 (원형 테이블 구조)
            
            # 탐색 중 아예 빈 슬롯을 만나면 더 이상 당겨올 데이터가 없으므로 종료
            if H[j] is unoccupied: 
                return key
            
            # 현재 j 위치에 있는 데이터의 원래 고향(원래 해시값 인덱스)을 구함
            k = f(H[j].key)
            
            # [핵심 조건 검사]
            # 데이터의 원래 고향(k)이 현재 비어있는 자리(i)보다 같거나 뒤에 있고,
            # 현재 검사 중인 자리(j)보다는 앞에 있다면 내부 루프 탈출
            if (k <= i <= j):  # (필기 노트의 인덱스 관계식 참고)
                break
        
        # 조건에 부합하는 j 위치의 데이터를 비어있는 i 위치로 당겨옴 (이사)
        H[i] = H[j]
        
        # 데이터가 빠져나간 j 자리가 새로운 빈 자리가 되므로 i를 j로 갱신
        i = j
```

---

## 3. 코드 라인별 상세 해설

* **`j = (j + 1) % m`**
  * 테이블의 맨 끝 인덱스에 도달했을 때 다시 `0`번 인덱스로 돌아오도록 만드는 **원형 구조(Circular)** 연산입니다. (`m`은 해시 테이블의 크기)
* **`k = f(H[j].key)`**
  * 조사 중인 데이터가 원래 충돌이 없었다면 들어가야 했을 **최초의 해시 주소**를 구합니다.
* **`if (k <= i <= j)` (이미지 필기: `k < i <= j`)**
  * 현재 비어있는 구멍 `i`를 기준으로, 뒤에 있는 데이터 `j`가 **'충돌 때문에 밀려나서 어쩔 수 없이 뒤에 가 있는 상태인가?'**를 판별하는 조건입니다. 
  * 이 조건이 `True`라면 데이터 `j`를 `i` 자리로 당겨와도 해시 탐색 경로가 깨지지 않습니다.

---

## 4. 요약
* **단순 삭제 금지**: 선형 조사법에서 중간 값을 그냥 지우면 뒤쪽 데이터 검색이 불가능해집니다.
* **데이터 이동**: 이 알고리즘은 데이터를 지운 후, 뒤쪽 데이터들의 원래 해시 주소(`k`)를 확인하여 **앞으로 당겨올 수 있는 데이터들을 차례대로 이사시키는 방식**으로 문제를 해결합니다.


## 5. 성능 요약 및 결론
*   **시간 복잡도**: 클러스터가 작을 때는 **평균 $O(1)$**을 유지하지만, 클러스터가 커질수록 성능이 저하됩니다.
*   선형 조사법은 구현이 쉽지만 클러스터링에 매우 취약하므로, 이를 개선하기 위해 이차 조사법이나 이중 해싱 같은 방법이 고안되었습니다.
