# 🕸️ [강의 자료] 그래프(Graph)의 기본 연산 및 성능 분석

그래프를 인접 행렬과 인접 리스트로 표현했을 때, 각각의 메모리 사용량과 주요 연산(탐색, 삽입, 삭제)의 효율성을 비교하여 학습합니다.

## 1. 그래프의 구성 요소 및 표기법
*   **정점(Vertex/Node)**: 정점의 집합을 $V$, 그 개수를 $n$이라 표기합니다.
*   **간선(Edge/Link)**: 정점 간의 연결 관계를 나타내는 간선의 집합을 $E$, 그 개수를 $m$이라 표기합니다.
*   **방향성**: 화살표가 있는 **방향 그래프**와 화살표가 없는 **무방향 그래프**로 나뉩니다.
*   **가중치(Weight)**: 간선에 비용이나 거리 같은 값을 부여할 수 있으며, 인접 행렬에서는 0/1 대신 해당 가중치 값을 저장합니다.

---

## 2. 메모리 사용량 비교
*   **인접 행렬 (Adjacency Matrix)**: $n \times n$ 2차원 배열을 사용하므로 항상 **$O(n^2)$**의 메모리가 필요합니다.
*   **인접 리스트 (Adjacency List)**: 노드 $n$개와 실제 존재하는 간선 $m$개의 정보만 저장하므로 **$O(n + m)$**의 메모리를 사용합니다.
    *   간선이 적은 **희소 그래프(Sparse Graph)**일수록 인접 리스트가 훨씬 효율적입니다.

---

## 3. 주요 연산별 시간 복잡도 분석

| 연산 내용 | 인접 행렬 | 인접 리스트 | 설명 |
| :--- | :---: | :---: | :--- |
| **두 노드의 인접 여부 확인** (`is_adjacent(u, v)`) | **$O(1)$** | $O(n)$ | 행렬은 인덱스로 즉시 확인 가능하나, 리스트는 해당 노드의 연결 리스트를 서치해야 함. |
| **특정 노드의 모든 이웃 확인** | $O(n)$ | **$O(\text{degree}(u))$** | 행렬은 $n$개 열을 다 검사해야 하지만, 리스트는 연결된 노드 수만큼만 루프를 돌면 됨. |
| **새로운 간선 삽입** (`insert_edge`) | **$O(1)$** | **$O(1)$** | 행렬은 값을 대입하고, 리스트는 맨 앞(`push_front`)이나 맨 뒤(`append`)에 추가함. |
| **기존 간선 삭제** (`delete_edge`) | **$O(1)$** | $O(n)$ | 행렬은 0으로 세팅하면 되나, 리스트는 해당 노드를 서치해서 지워야 함. |

---

## 4. 구현 및 선택 가이드

### 4.1 그래프 특성에 따른 선택
*   **희소 그래프(Sparse Graph)**: 간선 수가 노드 수에 비해 적은 경우. **인접 리스트**가 메모리와 이웃 방문 연산에서 유리합니다.
*   **밀집 그래프(Dense Graph)**: 모든 노드 사이에 간선이 거의 다 존재하는 경우. **인접 행렬**을 써도 메모리 낭비가 적고 탐색 속도가 빠릅니다.

### 4.2 파이썬 구현 팁
*   파이썬의 `list`를 사용하여 인접 리스트를 구현할 때, 새로운 간선을 삽입할 때는 `append()` 함수를 사용하는 것이 좋습니다. 
*   연결 리스트의 `push_front`와 마찬가지로 리스트의 맨 뒤에 추가하는 `append()`는 **상수 시간 $O(1)$**을 보장하기 때문입니다.
*   인접 리스트 내 노드들의 순서는 정렬될 필요가 없으므로 자유롭게 삽입해도 무방합니다.
