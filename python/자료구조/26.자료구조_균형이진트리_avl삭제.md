# 🌲 [강의 자료] AVL 트리의 삭제 연산과 파급 효과(Propagation)

AVL 트리의 삭제 연산은 삽입 연산보다 더 복잡한 과정을 거칩니다. 노드를 삭제한 후 무너진 균형을 회복하기 위해 루트 노드까지 거슬러 올라가며 연속적인 회전이 필요할 수 있기 때문입니다.

## 1. 삭제 연산의 기본 단계

AVL 트리의 삭제는 크게 세 단계로 진행됩니다.

1.  **노드 삭제**: 일반 이진 탐색 트리(BST)의 삭제 방식인 `delete_by_merging` 또는 `delete_by_copying`을 사용하여 노드를 제거합니다.
2.  **대상 노드 리턴**: 삭제 후 균형이 깨질 가능성이 있는 가장 깊은 곳의 노드(보통 삭제된 노드의 부모 노드)를 찾아 반환합니다.
3.  **루트까지 리밸런싱**: 반환된 노드부터 루트 노드에 도달할 때까지 위로 올라가며 각 레벨의 균형 상태를 점검하고 필요시 회전을 수행합니다.

---

## 2. 삭제 시 리밸런싱을 위한 노드 설정 ($z, y, x$)

삽입 연산과 달리, 삭제 연산에서는 균형이 깨진 노드에서 **더 높이가 높은(무거운) 쪽**으로 내려가며 회전 대상 노드들을 선정합니다.

*   **노드 $z$**: 삭제 발생 후 루트 방향으로 올라가며 처음으로 균형이 깨진(높이 차이가 2가 된) 조상 노드입니다.
*   **노드 $y$**: $z$의 자식 노드 중 **높이가 더 큰 쪽**의 노드입니다. (무거운 쪽에서 가벼운 쪽으로 높이를 나눠주기 위함).
*   **노드 $x$**: $y$의 자식 노드 중 **높이가 더 큰 쪽**의 노드입니다.

---

## 3. 삭제 연산의 핵심: 파급 효과 (Propagation)

AVL 트리 삭제가 삽입보다 까다로운 결정적인 이유는 **회전의 파급 효과** 때문입니다.

*   **삽입의 경우**: 한 번(또는 두 번)의 회전으로 해당 서브트리의 높이가 삽입 전과 동일해지므로, 그 이상의 조상 노드들은 영향을 받지 않습니다.
*   **삭제의 경우**: 특정 노드 $z$에서 회전을 통해 균형을 맞추더라도, 그 결과로 해당 서브트리의 전체 높이가 줄어들 수 있습니다. 이 높이 변화가 다시 그 부모 노드($w$)의 균형을 깨뜨릴 수 있으며, 이 현상은 **루트 노드까지 연쇄적으로 발생**할 수 있습니다.
*   **결과**: 최악의 경우 리프 노드에서 루트까지 모든 레벨(약 $\log n$번)에서 회전 연산이 발생할 수 있습니다.

---

## 4. 삭제 연산 의사코드 (Pseudo-code)

강의에서 설명된 삭제 및 리밸런싱 로직의 핵심 구조입니다.

```python
def delete(self, u):
    # 1. 기존 BST 삭제 로직 수행 및 영향을 받는 가장 깊은 노드 v 확보
    v = super().delete_by_copying(u) # 또는 delete_by_merging
    
    # 2. 루트 노드에 도달할 때까지 반복적으로 균형 확인
    while v is not None:
        if not self.is_balanced(v): # 균형이 깨졌다면 (v가 곧 z)
            z = v
            # 무거운 쪽 자식을 y와 x로 선택
            y = self.get_taller_child(z)
            x = self.get_taller_child(y)
            
            # 리밸런싱 수행 (1회 또는 2회 회전) 후 새로운 서브트리 루트 반환
            v = self.rebalance(z, y, x)
            
        # 3. 부모 노드로 이동하여 계속 확인 (파급 효과 처리)
        actual_last_node = v # 루트 갱신을 위해 마지막 노드 추적
        v = v.parent
        
    # 4. 회전으로 인해 루트가 바뀌었을 수 있으므로 루트 노드 업데이트
    self.root = actual_last_node
```

---

## 5. 성능 및 시간 복잡도 분석

| 단계 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| **탐색 및 삭제** | **$O(\log n)$** | 삭제할 노드를 찾고 물리적으로 제거하는 과정 |
| **리밸런싱 횟수** | **$O(\log n)$** | 최악의 경우 모든 레벨에서 회전 발생 (파급 효과) |
| **회전 1회 시간** | **$O(1)$** | 링크 수정은 상수 시간에 완료 |
| **총 시간 복잡도** | **$O(\log n)$** | 높이가 $\log n$으로 유지되므로 전체 연산은 로그 시간 보장 |

## 💡 요약
AVL 트리의 삭제는 노드 제거 후 **루트 방향으로 거슬러 올라가며 매 레벨마다 균형을 재점검**해야 합니다. 비록 회전 횟수는 삽입보다 많아질 수 있지만(최대 $\log n$번), 각 회전이 $O(1)$이므로 전체 수행 시간은 여전히 **$O(\log n)$**으로 매우 효율적입니다.
