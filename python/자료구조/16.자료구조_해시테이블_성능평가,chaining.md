### 1. 발전된 오픈 어드레싱(Open Addressing) 방식
선형 조사법(Linear Probing)의 단점인 클러스터(군집) 비대화 문제를 해결하기 위한 방법들입니다.

*   **이차 조사법 (Quadratic Probing)**: 충돌 시 한 칸씩 내려가는 대신, **보폭을 제곱($1^2, 2^2, 3^2 \dots$)으로 늘리며** 빈 슬롯을 찾습니다. 선형 조사법보다 클러스터 사이즈가 빨리 증가하지 않는 장점이 있지만, 삭제 연산 시 노드를 끌어올리는 로직이 더 복잡해집니다.
*   **이중 해싱 (Double Hashing)**: **두 개의 해시 함수**($f, g$)를 사용합니다. 첫 번째 해시값에서 충돌이 나면, 두 번째 해시값의 배수만큼($f(k)+g(k), f(k)+2g(k) \dots$) 건너뛰며 조사합니다. 해시 함수를 두 번 계산해야 하는 오버헤드는 있지만 효율적으로 동작합니다.

### 2. 해시 테이블의 성능 평가 지표
해시 테이블의 연산 속도($O(1)$)를 유지하기 위해 고려해야 할 요소들입니다.

*   **로드 팩터 (Load Factor, $\alpha = n/m$)**: 전체 슬롯 개수($m$) 대비 저장된 아이템 개수($n$)의 비율입니다. 로드 팩터가 1에 가까워질수록(즉, 빈칸이 없을수록) 충돌이 잦아지고 탐색 시간이 길어집니다.
*   **재해싱(Rehashing)과 성능 유지**: 평균적으로 **상수 시간($O(1)$)**을 유지하려면 항상 **일정 비율(예: 50% 이상)의 빈 슬롯을 유지**해야 합니다. 테이블이 너무 차면 다이내믹 어레이처럼 **2배 더 큰 테이블로 이사(모든 키를 다시 해싱)**하여 빈 공간을 확보합니다. 이 비용을 합쳐도 평균적으로는 $O(1)$에 동작합니다.

### 3. 체이닝 (Chaining) 방식
오픈 어드레싱과 달리, **각 슬롯에 여러 개의 아이템을 저장**할 수 있도록 설계된 방식입니다.

*   **구조**: 각 슬롯은 **한방향 연결 리스트(Singly Linked List)**의 헤드 역할을 합니다. 충돌이 발생하면 해당 슬롯의 리스트에 새 노드를 추가(`push_front`)하기만 하면 됩니다.
*   **연산 효율**:
    *   **삽입(Set)**: 리스트의 맨 앞에 넣으므로 항상 **$O(1)$**입니다.
    *   **탐색/삭제**: 해당 슬롯의 리스트를 처음부터 따라가며 확인해야 하므로, **리스트의 길이**에 비례하는 시간이 걸립니다.
*   **상수 시간 보장 조건**: **유니버설 해시 함수(Universal Hash Function)**를 사용하면, 슬롯당 평균 연결 리스트의 길이가 상수로 유지된다는 것이 증명되어 있습니다.

### 요약
해시 테이블은 **좋은 해시 함수(Universal Hash Function)**를 사용하고, **충분한 빈 슬롯을 유지**(오픈 어드레싱)하거나 **적절히 분산**(체이닝)시킨다면, 삽입·삭제·탐색 연산을 평균적으로 **상수 시간($O(1)$)** 내에 처리할 수 있는 매우 빠른 자료구조입니다.
