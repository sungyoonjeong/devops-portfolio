# 🕸️ [강의 자료] 그래프(Graph)의 정의와 표현법

그래프는 현실 세계의 복잡한 관계를 노드와 링크로 표현하는 가장 일반적이고 복잡한 자료구조입니다.

## 1. 그래프의 정의 및 구성 요소

### 1.1 기본 정의
그래프 $G$는 정점(Vertex)의 집합 $V$와 이들을 연결하는 간선(Edge)의 집합 $E$의 쌍으로 정의됩니다. $G = (V, E)$.

*   **정점 (Vertex/Node)**: 데이터를 담고 있는 개체입니다.
*   **간선 (Edge/Link)**: 두 정점 사이의 관계나 연결을 의미합니다.
    *   예: 정점 $V = \{1, 2, 3, 4, 5, 6\}$, 간선 $E = \{(1, 2), (2, 5), (1, 5), \dots\}$.

### 1.2 그래프의 예시
*   **언어/국가 관계**: 언어 간의 영향력이나 유사 관계 표현.
*   **지하철 노선도**: 역(노드)과 노선(간선)의 연결.
*   **항공 노선도**: 도시(노드)와 운항 노선(간선). 운항 빈도에 따라 간선의 굵기(가중치)를 다르게 표현 가능.

---

## 2. 주요 용어 정리

*   **인접성 (Adjacency)**: 두 노드가 하나의 간선으로 직접 연결되어 있을 때, 두 노드는 서로 **인접(Adjacent)**하다고 합니다.
*   **차수 (Degree)**: 한 노드에 연결된 간선의 개수입니다. 그래프 전체의 차수는 각 노드 차수 중 최댓값을 의미합니다.
*   **경로 (Path)**: 한 노드에서 다른 노드로 가는 간선들의 나열입니다. 이때 경로상의 노드들은 모두 달라야 합니다.
*   **사이클 (Cycle)**: 특정 노드에서 출발하여 다시 자기 자신으로 돌아오는 닫힌 경로입니다.
*   **트리와의 관계**: 트리는 **사이클이 없는 그래프**입니다. 따라서 트리 내의 두 노드를 연결하는 경로는 오직 하나뿐입니다.

---

## 3. 그래프의 종류

1.  **무방향 그래프 (Undirected Graph)**: 간선에 방향이 없어 양방향으로 이동이 가능합니다.
2.  **방향 그래프 (Directed Graph)**: 간선에 화살표로 방향성이 존재하며, 정해진 방향으로만 이동할 수 있습니다.
3.  **가중치 그래프 (Weighted Graph)**: 간선마다 비용, 거리, 시간 등의 수치(Weight)가 부여된 그래프입니다.

---

## 4. 그래프의 표현 방법 (Representation)

그래프를 코드로 저장하는 방식은 크게 두 가지입니다.

### 4.1 인접 행렬 (Adjacency Matrix)
$n \times n$ 크기의 2차원 배열을 사용하여 연결 관계를 표현합니다.
*   **방법**: 노드 $i$와 $j$가 연결되어 있으면 $A[i][j] = 1$, 아니면 $0$으로 표시합니다.
*   **특징**: 
    *   무방향 그래프의 경우 대각선을 기준으로 대칭 구조를 가집니다.
    *   가중치 그래프라면 $1$ 대신 가중치 값을 저장합니다.
*   **단점**: 노드 수($n$)에 비해 간선 수가 적어도 $n^2$의 메모리를 사용하므로 낭비가 심할 수 있습니다.

### 4.2 인접 리스트 (Adjacency List)
각 노드마다 자신과 인접한 노드들을 연결 리스트(Linked List)로 관리하는 방식입니다.
*   **방법**: 각 정점에 대해 해당 정점과 연결된 이웃 노드들의 ID를 리스트로 저장합니다.
*   **장점**: 존재하는 간선 정보($E$)만 저장하므로 메모리 사용이 효율적입니다.
*   **가중치 표현**: 리스트의 각 노드에 노드 ID와 함께 가중치 값을 멤버로 추가하여 저장합니다.
