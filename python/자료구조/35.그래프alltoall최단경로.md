# [강의 자료] 모든 쌍 최단 경로와 플로이드-워셜 알고리즘

이 자료는 그래프의 모든 정점 사이의 최단 거리를 구하는 방법과 그 핵심 원리인 플로이드-워셜 알고리즘에 대해 다룹니다.

## 1. 문제의 정의: "어디서 어디로 갈 것인가?"

우리가 알고 있는 최단 경로 문제에는 크게 두 가지 유형이 있습니다.

*   **단일 출발점 최단 경로 (Single-Source Shortest Path):** 특정 하나의 노드(s)에서 시작하여 다른 모든 노드로 가는 최단 경로를 구하는 문제입니다. 대표적으로 다익스트라(Dijkstra)와 벨만-포드(Bellman-Ford) 알고리즘이 있습니다.
*   **모든 쌍 최단 경로 (All-Pairs Shortest Path, All-to-All):** 그래프 상의 모든 노드 쌍 $(i, j)$에 대해 $i$에서 $j$로 가는 최단 경로를 모두 구하는 문제입니다.

---

## 2. 가장 단순한 접근법: 다익스트라 반복하기

가장 쉬운 해결 방법은 다익스트라 알고리즘을 모든 노드에 대해 한 번씩 수행하는 것입니다.

*   **방법:** 노드가 $n$개라면, 각 노드를 시작점으로 설정하여 다익스트라를 총 $n$번 호출합니다.
*   **시간 복잡도:** 
    *   일반적인 바이너리 힙을 사용한 다익스트라의 복잡도는 $O(m \log n)$입니다 ($m$은 간선 수, $n$은 정점 수).
    *   이를 $n$번 반복하면 총 $O(nm \log n)$이 걸립니다.
    *   만약 간선이 매우 많은 완전 그래프라면 $m$이 $n^2$에 가까워지므로, 최대 $O(n^3 \log n)$까지 늘어날 수 있습니다.

---

## 3. 플로이드-워셜 알고리즘: 동적 계획법(DP)의 활용

더 효율적이고 직관적인 방법이 바로 **플로이드-워셜 알고리즘**입니다. 이 알고리즘은 **동적 계획법(Dynamic Programming, DP)** 원리를 사용합니다.

### 핵심 아이디어: "중간에 누구를 거쳐갈 것인가?"
두 노드 $i$와 $j$ 사이의 최단 경로는 다음 두 가지 경우 중 하나입니다.
1.  중간에 특정 노드 $k$를 **거치지 않는** 경로
2.  중간에 특정 노드 $k$를 **거쳐가는** 경로

이 원리를 이용해 $1$번 노드부터 $n$번 노드까지 차례대로 "중간 노드"로 사용해 보며 경로를 업데이트합니다.

### DP 점화식
$d_{ij}^{(k)}$를 "1번부터 $k$번까지의 노드들만 중간 거점으로 사용하여 $i$에서 $j$로 가는 최단 거리"라고 정의합시다.

$$d_{ij}^{(k)} = \min(d_{ij}^{(k-1)}, d_{ik}^{(k-1)} + d_{kj}^{(k-1)})$$

*   $d_{ij}^{(k-1)}$: 노드 $k$를 거치지 않는 기존의 최단 거리.
*   $d_{ik}^{(k-1)} + d_{kj}^{(k-1)}$: 노드 $k$를 거쳐서 가는 새로운 경로의 거리.
*   이 두 값 중 더 작은 값을 선택하여 최단 거리를 갱신합니다.

---

## 4. 알고리즘 동작 단계 (Implementation)

### [Step 1] 테이블 초기화
먼저 $n \times n$ 크기의 2차원 리스트(테이블)를 만듭니다.
1.  **자기 자신으로 가는 경로:** $d_{ii} = 0$으로 설정합니다.
2.  **간선이 존재하는 경우:** 해당 간선의 가중치(Weight)를 넣습니다.
3.  **간선이 없는 경우:** 무한대(Infinity, $\infty$)로 설정합니다.

### [Step 2] 3중 반복문을 통한 갱신
가장 바깥쪽 반복문은 중간 노드($k$)를 결정하고, 안쪽 두 반복문은 모든 출발지($i$)와 도착지($j$) 쌍을 검사합니다.

```python
# 파이썬 예시 코드
for k in range(1, n + 1):          # 중간 노드 k
    for i in range(1, n + 1):      # 출발 노드 i
        for j in range(1, n + 1):  # 도착 노드 j
            # i에서 j로 바로 가는 것보다 k를 거쳐가는 게 더 빠르다면 갱신
            if d[i][j] > d[i][k] + d[k][j]:
                d[i][j] = d[i][k] + d[k][j]
```


---

## 5. 복잡도 및 특징 분석

*   **시간 복잡도:** 3중 반복문을 사용하므로 정점 수 $n$에 대해 **$O(n^3)$**입니다. 다익스트라를 $n$번 반복하는 $O(n^3 \log n)$보다 로그 인자만큼 더 빠릅니다.
*   **공간 복잡도:** $n \times n$ 크기의 테이블이 필요하므로 **$O(n^2)$**입니다.
*   **장점:** 구현이 매우 간단하며, 간선의 가중치가 **음수**인 경우에도 사용할 수 있습니다 (단, 음수 사이클이 없어야 함).

---

### 요약
플로이드-워셜 알고리즘은 "중간 정점 $k$를 거쳐가는 것이 더 이득인가?"를 모든 노드 쌍에 대해 반복적으로 확인하여 모든 최단 경로를 찾아내는 강력한 도구입니다.
