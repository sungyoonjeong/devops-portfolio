# 🌲 [강의 자료] 레드-블랙 트리(Red-Black Tree)의 이해와 성질

레드-블랙 트리는 가장 유명하고 널리 사용되는 **균형 이진 탐색 트리(Balanced BST)** 중 하나입니다. 노드에 색상을 부여하여 트리의 균형을 유지함으로써 탐색, 삽입, 삭제 연산에서 최악의 경우에도 효율적인 속도를 보장합니다.

## 1. 레드-블랙 트리의 노드 구조

레드-블랙 트리는 일반적인 이진 트리와 노드를 다루는 방식에서 차이가 있습니다.

*   **노드 색상**: 이름에서 알 수 있듯이 모든 노드는 **레드(Red)** 또는 **블랙(Black)** 중 하나의 색상을 가집니다.
*   **NIL 노드 (External Node)**: 파이썬의 `None`이나 C의 `NULL`에 해당하는 빈 노드를 레드-블랙 트리에서는 **'NIL 노드'** 혹은 **'리프 노드'**라는 독립된 노드로 취급합니다.
*   **내부 노드 (Internal Node)**: 실제 데이터를 담고 있는 노드를 말하며, 모든 내부 노드는 자식으로 반드시 두 개의 리프 노드(NIL)를 가집니다.

## 2. 레드-블랙 트리의 5가지 조건

어떤 이진 탐색 트리가 레드-블랙 트리가 되기 위해서는 다음의 5가지 조건을 반드시 만족해야 합니다.

1.  **노드 색상**: 모든 노드는 레드 아니면 블랙이어야 합니다.
2.  **루트 성질**: 루트 노드는 항상 **블랙**이어야 합니다.
3.  **리프 성질**: 모든 리프 노드(NIL 노드)는 **블랙**이어야 합니다.
4.  **레드 연속 금지**: 레드 노드의 자식 노드는 반드시 **블랙**이어야 합니다 (즉, 레드 노드가 연속해서 나타날 수 없습니다).
5.  **블랙 높이 일치**: 임의의 노드에서 그 자손인 리프 노드로 가는 모든 경로에는 **동일한 개수의 블랙 노드**가 있어야 합니다.

이 중 **5번 조건**은 트리의 높이를 $O(\log n)$으로 제한하는 데 결정적인 역할을 하는 가장 중요한 성질입니다.

## 3. [심화] 높이의 증명: 왜 $O(\log n)$인가?

레드-블랙 트리에 $n$개의 내부 노드가 있을 때, 트리의 높이($h$)가 $2\log_2(n+1)$을 넘지 않음을 수학적으로 증명할 수 있습니다.

### 3.1 블랙 높이(Black Height, $bh$)의 정의
*   **$bh(v)$**: 노드 $v$에서 리프 노드까지 내려가는 경로상에 있는 **블랙 노드의 개수**입니다. 단, **노드 $v$ 자신의 색상은 제외**하고 계산합니다.

### 3.2 내부 노드 개수에 대한 정리
> **정리**: 노드 $v$를 루트로 하는 서브트리는 적어도 **$2^{bh(v)} - 1$개**의 내부 노드를 포함합니다.

**증명 (높이에 대한 귀납법)**:
1.  **Base Case**: 노드 $v$의 높이가 0인 경우(리프 노드), $bh(v)=0$이고 내부 노드 개수는 $2^0 - 1 = 0$이므로 성립합니다.
2.  **Inductive Step**: 높이가 $k$ 미만일 때 성립한다고 가정하고, 높이가 $k$인 노드 $v$를 봅니다.
    *   $v$의 두 자식 노드 $w$와 $z$의 블랙 높이는 $v$가 레드면 $bh(v)$, 블랙이면 $bh(v)-1$입니다.
    *   결과적으로 자식들의 블랙 높이는 최소 $bh(v)-1$ 이상입니다.
    *   전체 내부 노드 개수는 $(자식\ 1의\ 노드) + (자식\ 2의\ 노드) + 1(자기\ 자신)$이므로, $(2^{bh(v)-1}-1) + (2^{bh(v)-1}-1) + 1 = 2^{bh(v)}-1$이 되어 성립합니다.

### 3.3 최종 높이 유도
*   조건 4(레드 연속 금지)에 의해, 루트에서 리프까지의 경로에서 블랙 노드의 수는 전체 높이의 **절반 이상**이어야 합니다.
*   따라서 루트의 블랙 높이 $bh(root) \geq h/2$입니다.
*   앞선 정리에 대입하면, 내부 노드 개수 $n \geq 2^{h/2} - 1$이 됩니다.
*   이를 $h$에 대해 정리하면 **$h \leq 2\log_2(n+1)$**이 되어, 높이가 로그 단위로 제한됨이 증명됩니다.

## 4. 요약 및 결론

*   **균형의 원리**: 레드-블랙 트리는 어떤 경로로 내려가도 만나는 블랙 노드 수를 일정하게 유지함으로써 트리가 한쪽으로 치우치는 것을 방지합니다.
*   **성능**: 이 5가지 조건을 유지하면 트리의 높이가 항상 $O(\log n)$으로 유지되므로, 매우 효율적인 탐색이 가능해집니다.
*   **실제 활용**: 삽입과 삭제 연산 시 이 조건들을 깨뜨리지 않기 위해 색상을 바꾸거나 회전(Rotation)을 수행하며, 이는 실제 시스템에서 매우 복잡하지만 강력하게 동작합니다.
