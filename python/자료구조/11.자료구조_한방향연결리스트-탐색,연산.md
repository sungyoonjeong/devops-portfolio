# 📖 강의: 한방향 연결 리스트의 탐색과 제너레이터

이 강의에서는 연결 리스트의 데이터를 효율적으로 찾고, 파이썬의 `for` 루프에서 자연스럽게 사용할 수 있게 만드는 방법을 배웁니다.

### 1. 탐색(Search) 연산
탐색은 리스트 내에 특정 키(key) 값을 가진 노드가 있는지 확인하고, 해당 노드를 반환하는 연산입니다.

*   **작동 원리**:
    1.  연결 리스트에서 위치를 알 수 있는 유일한 노드인 **헤드(Head) 노드부터 출발**합니다.
    2.  현재 노드(`v`)가 `None`이 아닐 때까지 반복하며, 노드의 키 값과 찾고자 하는 키 값을 비교합니다.
    3.  **일치하는 경우**: 해당 노드(`v`)를 즉시 반환합니다.
    4.  **일치하지 않는 경우**: 다음 노드(`v.next`)로 이동하여 탐색을 계속합니다.
    5.  **끝까지 못 찾은 경우**: 리스트의 끝(`None`)에 도달하면 `None`을 반환하여 값이 없음을 알립니다.
*   **시간 복잡도**: 최악의 경우 리스트의 모든 노드($n$개)를 다 확인해야 하므로 **$O(n)$**이 소요됩니다.

### 2. 제너레이터(Generator) 구현
파이썬의 일반 리스트처럼 `for x in L:`과 같은 **순회(Iteration)** 문법을 연결 리스트에서도 사용하기 위해 제너레이터를 만듭니다.

*   **`__iter__` 스페셜 메소드**: 
    클래스 내부에 `__iter__`라는 이름의 함수를 선언하면, 이 리스트는 '반복 가능한 객체'가 됩니다. 구현 방식은 탐색 연산과 유사하게 헤드부터 차례로 방문하는 구조입니다.
*   **`yield` 키워드의 역할**:
    *   `return`과 비슷하지만, 값을 반환한 뒤 함수를 종료하지 않고 **그 자리에서 실행을 일시 정지**합니다.
    *   `for` 루프가 다음 값을 요청하면, 멈췄던 지점(보통 `v = v.next`)부터 다시 실행을 재개합니다.
*   **종료 메커니즘**:
    모든 노드를 방문하여 `v`가 `None`이 되면 함수가 종료됩니다. 이때 파이썬 내부적으로 **`StopIteration`**이라는 메시지가 발생하여 `for` 루프가 안전하게 끝납니다.

### 3. 요약 및 장점
*   **탐색**을 통해 리스트 내 특정 데이터의 위치를 파악할 수 있습니다.
*   **제너레이터**를 구현하면 `while` 루프를 복잡하게 쓰지 않고도, **`for...in` 문법을 통해 리스트의 모든 노드를 우아하고 간결하게 처리**할 수 있습니다.

### 1. 탐색(Search) 연산 코드,
특정 키 값을 찾아 해당 노드를 반환하는 함수입니다.

```python
def search(self, key):
    v = self.head              # 헤드 노드부터 출발합니다
    while v is not None:       # 노드가 None이 아닐 때까지 반복합니다
        if v.key == key:       # 현재 노드의 키가 찾는 값과 같다면
            return v           # 해당 노드를 반환합니다
        v = v.next             # 일치하지 않으면 다음 노드로 이동합니다
    return None                # 끝까지 찾지 못하면 None을 반환합니다
```

### 2. 제너레이터(`__iter__`) 코드
`for` 루프에서 리스트를 사용할 수 있게 해주는 특별한 메소드입니다.

```python
def __iter__(self):
    v = self.head              # 순회를 위해 헤드 노드부터 시작합니다
    while v is not None:       # 리스트의 끝에 도달할 때까지 반복합니다
        yield v                # 현재 노드를 반환하고 잠시 멈춥니다(핵심 키워드),
        v = v.next             # 다시 호출되면 다음 노드로 이동합니다
```

### 💡 강의 내 코드 설명의 특징
*   **탐색 연산**: 리스트의 모든 노드를 차례대로 확인해야 하므로 최악의 경우 **$O(n)$**의 시간이 걸린다는 점을 강조합니다.
*   **제너레이터**: `yield` 키워드를 사용하면 `return`과 달리 함수가 종료되지 않고 상태를 유지하며, 모든 순회가 끝나면 파이썬이 내부적으로 `StopIteration` 메시지를 발생시켜 루프를 종료한다는 메커니즘을 설명합니다,.
