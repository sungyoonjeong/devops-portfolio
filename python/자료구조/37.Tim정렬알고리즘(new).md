# [강의 자료] 실전 최강의 정렬 알고리즘: Tim Sort (팀 정렬)

이 자료는 파이썬, 자바, 안드로이드 등 현대 프로그래밍 언어의 표준 정렬 라이브러리에서 가장 많이 사용되는 **Tim Sort**의 원리와 최적화 기법을 초보자 눈높이에서 설명합니다.

---

## 1. Tim Sort란 무엇인가?

*   **탄생:** 2002년 팀 피터스(Tim Peters)가 파이썬의 정렬 함수(`sort()`)를 위해 만들었습니다.
*   **정체:** **삽입 정렬(Insertion Sort)**과 **병합 정렬(Merge Sort)**을 아주 영리하게 결합한 '하이브리드(Hybrid) 정렬'입니다.
*   **철학:** "실제 세상의 데이터는 완전히 랜덤하지 않다. 이미 어느 정도 정렬된 부분이 반드시 존재한다"는 점을 이용합니다.
*   **사용처:** 파이썬, 자바, 자바스크립트, 스위프트, 안드로이드 등 거의 모든 현대 시스템의 기본 정렬 알고리즘입니다.

---

## 2. 왜 두 알고리즘을 섞었을까?

Tim Sort는 각 알고리즘의 단점은 버리고 장점만 취했습니다.

### ① 삽입 정렬 (Insertion Sort)
*   **특징:** 하나씩 제자리를 찾아 삽입하는 방식.
*   **장점:** 정렬할 개수가 적을 때 매우 빠르며, 이미 정렬된 데이터라면 비교만 하고 끝나기에 $O(n)$의 최고 성능을 냅니다.
*   **단점:** 데이터가 많아지면 이동 횟수가 급증하여 $O(n^2)$으로 느려집니다.

### ② 병합 정렬 (Merge Sort)
*   **특징:** 반씩 쪼갠 뒤 다시 합치면서 정렬하는 방식.
*   **장점:** 데이터가 아무리 많아도 일정한 성능($O(n \log n)$)을 보장합니다.
*   **단점:** 추가적인 메모리 공간이 필요(Not In-place)하다는 단점이 있습니다.

---

## 3. Tim Sort의 핵심 동작 과정

### [Step 1] 런(Run) 만들기
전체 데이터를 처음부터 읽으며 **이미 정렬된 구간**을 찾습니다. 이 구간을 **'런(Run)'**이라고 부릅니다.
*   런의 크기가 너무 작으면(보통 32~64개 미만), **삽입 정렬**을 사용해 강제로 런의 길이를 채웁니다.
*   내림차순으로 정렬된 구간은 단순히 뒤집어서 오름차순으로 만듭니다.

### [Step 2] 스택(Stack)을 이용한 지능적 병합
만들어진 런들을 무작정 합치지 않고, **스택**에 넣어 관리하며 특정 조건이 맞을 때만 병합합니다.

*   **병합 조건 (가장 위의 런 A, B, C):**
    1.  $|B| > |A|$ (두 번째 런이 첫 번째보다 커야 함)
    2.  $|C| > |B| + |A|$ (세 번째 런이 위쪽 두 개의 합보다 커야 함)
*   **이유:** 이 조건을 유지하면 스택에 남은 런들의 크기가 **피보나치 수열**처럼 커지게 되어, 비슷한 크기의 런끼리 합칠 수 있고 스택 크기도 $O(\log n)$으로 작게 유지됩니다.

---

## 4. 초보자를 위한 고급 최적화 기법

### ① 이진 탐색 삽입 정렬 (Binary Insertion Sort)
삽입 정렬을 할 때, 들어갈 위치를 앞에서부터 하나씩 찾는 대신 **이진 탐색(Binary Search)**을 사용합니다.
*   **효과:** 비교 횟수를 $O(n^2)$에서 $O(n \log n)$ 수준으로 대폭 줄여줍니다.

### ② 추가 메모리 절약 (Memory Optimization)
일반 병합 정렬은 전체 데이터 크기($n$)만큼의 추가 메모리가 필요하지만, Tim Sort는 **더 작은 쪽의 런 크기**만큼만 메모리를 할당합니다.
*   **방식:** 작은 런을 임시 공간에 복사한 뒤, 원본 리스트에 직접 덮어쓰며 병합합니다.

---

## 5. 최종 성능 분석

*   **시간 복잡도:**
    *   **최악의 경우:** $O(n \log n)$
    *   **최선의 경우 (이미 정렬된 경우):** $O(n)$
*   **공간 복잡도:** $O(n)$ (추가 메모리 사용)
*   **안정성(Stable):** 같은 값의 순서가 바뀌지 않는 '안정 정렬'입니다.

---

### 💡 요약하자면
Tim Sort는 **"작은 구간은 삽입 정렬로 빠르게, 큰 구간은 병합 정렬로 안정적으로"** 처리하면서, **"이미 정렬된 실생활 데이터의 특성"**을 극한으로 이용하는 아주 똑똑한 알고리즘입니다.
