# 🌲 [강의 자료] 균형 이진 탐색 트리(Balanced BST)와 회전 연산

이진 탐색 트리(BST)는 탐색에 효율적이지만, 입력되는 데이터의 순서에 따라 성능이 크게 좌우됩니다. 이를 해결하기 위한 **균형 이진 탐색 트리**의 정의와 트리의 균형을 맞추는 핵심 기법인 **회전(Rotation)**에 대해 학습합니다.

## 1. 균형 이진 탐색 트리(Balanced BST)의 필요성

### 1.1 이진 탐색 트리의 한계
*   **탐색 시간의 결정**: BST에서 탐색, 삽입, 삭제 연산의 비용은 트리의 높이($h$)에 비례하여 **$O(h)$**의 시간이 걸립니다.
*   **최악의 경우 ($O(n)$)**: 데이터가 정렬된 순서(예: 1, 10, 8, 5, 7...)로 삽입되면 트리가 한쪽으로 치우친 연결 리스트 형태가 되어, 높이 $h$가 $n-1$까지 증가할 수 있습니다.

### 1.2 균형 이진 탐색 트리의 정의
*   **개념**: $n$개의 노드가 저장된 이진 탐색 트리 중에서, 트리의 높이가 항상 **$\log n$에 비례하는 정도**로 유지되도록 강제하는 트리입니다.
*   **대표적인 종류**:
    *   **AVL 트리**: 1962년에 제안된 최초의 균형 BST.
    *   **레드-블랙 트리 (Red-Black Tree)**: 가장 널리 사용되는 형태.
    *   **2-3-4 트리**, **스플레이 트리 (Splay Tree)** 등.

---

## 2. 핵심 조정 연산: 회전(Rotation)

트리의 높이가 특정 기준을 넘어 커지려고 할 때, 트리의 일부를 **회전**시켜 높이를 낮게 유지합니다.

### 2.1 우회전 (Right Rotation)
노드 $Z$를 중심으로 오른쪽 방향으로 회전시키는 연산입니다 (주로 왼쪽 서브트리가 너무 깊을 때 수행).

*   **작동 원리**:
    1.  $Z$의 왼쪽 자식인 $X$가 $Z$의 위치로 올라옵니다.
    2.  기존의 $Z$는 $X$의 오른쪽 자식이 되어 한 레벨 내려갑니다.
    3.  $X$의 원래 오른쪽 자식이었던 서브트리 $B$는 $Z$의 왼쪽 자식으로 옮겨집니다.
*   **결과**: 왼쪽 서브트리의 높이는 1 레벨 줄어들고, 오른쪽 서브트리의 높이는 1 레벨 늘어나 전체적인 균형이 맞춰집니다.

### 2.2 좌회전 (Left Rotation)
노드 $X$를 중심으로 왼쪽 방향으로 회전시키는 연산입니다. 우회전과 정확히 **대칭적**으로 동작합니다.

## 3. 회전 연산의 구현 (의사코드)

회전 연산은 총 **6개의 링크**를 업데이트함으로써 완료됩니다.

```python
def rotateRight(self, z):
    if z is None: return
    x = z.left
    if x is None: return # x가 없으면 회전 불가
    
    b = x.right # x의 오른쪽 서브트리
    
    # 1. x를 z의 부모와 연결 (z가 있던 위치로 x를 올림)
    x.parent = z.parent
    if z.parent is not None:
        if z.parent.left == z:
            z.parent.left = x
        else:
            z.parent.right = x
    
    # 2. x의 오른쪽 자식으로 z를 연결
    x.right = z
    z.parent = x
    
    # 3. z의 왼쪽 자식으로 x의 원래 오른쪽 자식(b)을 연결
    z.left = b
    if b is not None:
        b.parent = z
        
    # 4. 만약 z가 루트였다면 x를 새로운 루트로 설정
    if self.root == z:
        self.root = x
```
---

## 4. 성능 분석 및 결론

*   **시간 복잡도**: 회전 연산은 단순히 고정된 개수(6개)의 링크만 변경하면 되므로, 트리의 크기와 상관없이 항상 **$O(1)$(상수 시간)** 내에 수행됩니다.
*   **결론**: 이 $O(1)$ 시간의 회전 연산을 적절히 사용하여 트리의 높이를 $\log n$으로 유지함으로써, 이진 탐색 트리의 모든 기본 연산($O(h)$)을 최종적으로 **$O(\log n)$**에 보장할 수 있게 됩니다.
