# [강의 자료] 파이썬 딕셔너리(dict)는 어떻게 동작하는가?

이 자료는 파이썬에서 가장 자주 사용되는 자료구조 중 하나인 **딕셔너리(Dictionary)**가 내부적으로 어떻게 데이터를 저장하고, 왜 그렇게 빠른지 그 비밀을 파헤칩니다.

## 1. 딕셔너리의 기본 구조: 키(Key)와 값(Value)

*   **정의:** 딕셔너리는 **키(Key)**와 **값(Value)**의 쌍을 저장하는 자료구조입니다.
*   **예시:** 학번(Key)과 학생 이름(Value)의 관계와 같습니다. 학번은 중복되지 않는 고유한 값이며, 이를 통해 학생 정보를 바로 찾을 수 있습니다.
*   **실제 구현:** 파이썬 내부에서 딕셔너리는 **해시 테이블(Hash Table)**이라는 구조로 구현되어 있습니다. 해시 테이블은 기본적으로 1차원 리스트(배열) 형태를 띱니다.

---

## 2. 해시 함수(Hash Function)와 인덱스 결정

데이터를 해시 테이블에 저장하려면 '몇 번째 칸(슬롯)'에 넣을지 결정해야 합니다.

1.  **해시 함수:** 키값을 입력받아 아주 큰 정수(해시값)로 변환합니다.
2.  **인덱스 계산:** 생성된 해시값을 현재 테이블의 크기로 나눈 **나머지(%)**를 취해 실제 저장될 슬롯 번호(인덱스)를 정합니다.
    *   예: 해시값이 12345이고 테이블 크기가 8이라면, `12345 % 8 = 1`이 되어 1번 슬롯에 저장됩니다.

---

## 3. 충돌(Collision)과 해결 방법

서로 다른 두 키가 해시 함수를 거쳤을 때 하필 **같은 인덱스**가 나오는 상황을 **충돌(Collision)**이라고 합니다. 파이썬은 이를 **오픈 어드레싱(Open Addressing)** 방식으로 해결합니다.

*   **오픈 어드레싱:** 충돌이 발생하면 비어 있는 다른 슬롯을 찾아가는 방식입니다.
*   **파이썬의 독특한 탐색 공식:** 단순히 다음 칸을 보는 것이 아니라, 아래와 같은 공식을 사용하여 다음 슬롯을 찾습니다.
    *   `다음 인덱스 = (5 * 현재 인덱스 + 1 + perturb) % 테이블 크기`
*   **무작위성(perturb):** `perturb`라는 변수에 키의 해시값을 넣어 초반에는 무작위로 점프하며 빈 곳을 찾게 합니다. 루프를 돌 때마다 이 값은 5비트씩 오른쪽으로 이동(Shift)하여 결국 0이 됩니다.
*   **결과:** 이 공식은 테이블 크기가 2의 거듭제곱(2, 4, 8, 16...)일 때, 모든 슬롯을 중복 없이 한 번씩 다 방문할 수 있음을 보장합니다.

---

## 4. 테이블 크기 조정 (Resizing)

데이터가 계속 추가되어 슬롯이 꽉 차면 더 이상 저장할 수 없게 됩니다.

*   **초기 크기:** 파이썬 딕셔너리는 처음 생성될 때 **8개**의 슬롯으로 시작합니다.
*   **확장 기준:** 테이블의 **약 2/3(66%) 이상**이 채워지면, 현재 크기의 **2배**인 더 큰 테이블을 만들어 모든 데이터를 이사(Rehashing)시킵니다.
*   **효율성:** 항상 약 1/3의 빈 슬롯을 유지함으로써 충돌 확률을 낮추고 빠른 속도를 유지합니다.

---

## 5. 성능 분석: 왜 딕셔너리는 빠른가?

*   **평균 시간 복잡도:** 삽입(Set), 삭제(Remove), 탐색(Search) 연산 모두 평균적으로 **$O(1)$ (상수 시간)**이 걸립니다.
*   **분할 상환 시간(Amortized Time):** 가끔 발생하는 리사이징(이사) 시에는 시간이 오래 걸리지만, 이를 전체 연산 횟수로 나누어 평균을 내면 결국 $O(1)$의 매우 빠른 속도를 보장한다는 논리입니다.

---

### 💡 요약
파이썬 딕셔너리는 **해시 함수**를 통해 데이터를 즉시 찾고, **충돌 시 독특한 수식**으로 빈 곳을 효율적으로 탐색하며, **적절한 시기에 크기를 키워** 최상의 성능을 유지하는 아주 똑똑한 자료구조입니다.
