# [강의 자료] 파이썬 리스트의 비밀: 동적 배열과 효율적인 사용법

이 자료는 파이썬에서 가장 많이 쓰이는 자료구조인 **리스트(List)**가 내부적으로 어떻게 동작하는지, 왜 데이터가 늘어나도 빠른지, 그리고 어떻게 사용하는 것이 가장 효율적인지 다룹니다.

## 1. 파이썬 리스트 vs C 언어 배열

파이썬 리스트는 내부적으로 **배열(Array)**의 성질을 갖지만, C 언어와 같은 전통적인 배열과는 저장 방식에 차이가 있습니다.

*   **C 언어 배열:** 메모리 상에 데이터(예: 정수 4바이트)를 **연속적으로 직접 저장**합니다. 인덱스를 알면 주소 계산(시작 주소 + 인덱스 × 크기)을 통해 한 번에 접근($O(1)$)할 수 있습니다.
*   **파이썬 리스트:** 데이터 자체가 아닌, 데이터가 저장된 **객체의 주소(포인터)**를 연속적으로 저장합니다. 
    *   이 때문에 파이썬 리스트는 **2단계 구조**를 가집니다 (주소 확인 → 실제 객체 방문).
    *   C 배열보다 조금 더 시간이 걸리지만, 여전히 인덱스를 통한 접근은 **상수 시간($O(1)$)**에 가능합니다.

---

## 2. 동적 배열(Dynamic Array)과 리사이징

파이썬 리스트는 크기가 고정되지 않고 필요에 따라 늘어나는 **동적 배열**입니다.

*   **메모리 오버헤드:** 빈 리스트를 만들어도 이를 관리하기 위한 정보(사이즈, 저장된 개수 등) 때문에 약 **56바이트**의 기본 메모리를 사용합니다.
*   **리사이징(Resizing) 원리:** 
    1.  데이터를 `append` 할 때 저장 공간이 꽉 차면, 기존보다 더 큰 새로운 메모리 공간을 할당받습니다.
    2.  기존 데이터를 새 공간으로 모두 복사(**이동 연산**)한 뒤, 새 데이터를 추가합니다.
    3.  파이썬은 효율성을 위해 현재 용량의 약 **1.125배**씩 공간을 키웁니다.

---

## 3. 분할 상환 수행 시간 (Amortized Time Complexity)

데이터를 추가할 때마다 리사이징이 일어나면 매우 느리겠지만, 파이썬 리스트의 `append`는 평균적으로 매우 빠릅니다. 이를 **분할 상환 분석**으로 설명합니다.

*   **개념:** 가끔 발생하는 비싼 비용(리사이징/이사 비용)을 평소의 저렴한 연산들에 골고루 나눠서 평균을 내는 방식입니다.
*   **월세 비유:** 매일 조금씩 돈을 모아 한 달에 한 번 월세를 내는 것과 같습니다. 마지막 날 큰돈이 나가지만, 하루 평균 비용은 일정합니다.
*   **증명 (Charging Method):** 
    *   리사이징 시 발생하는 $n$개의 데이터 이동 비용을 앞선 연산들에 **2원씩 추가 비용**을 걷어 적립해둔다고 가정합니다.
    *   이렇게 하면 리사이징이 발생했을 때 미리 모아둔 비용으로 충당이 가능하며, 결국 하나의 `append` 당 지불하는 총 비용은 **상수(약 3원)**로 수렴합니다.
    *   따라서 `append`의 평균 시간 복잡도는 **$O(1)$**입니다.

---

## 4. 리스트를 생성하는 4가지 방법의 효율성 비교

천 개의 원소를 가진 리스트를 만들 때, 방법에 따라 속도 차이가 크게 납니다.

1.  **더하기 연산(`+`):** `a = a + [i]` 방식. 매번 새로운 리스트를 복사하므로 가장 느립니다 (**12초**).
2.  **`append()` 사용:** 빈 리스트에 하나씩 추가. 동적 배열의 효율성을 이용하므로 빠릅니다 (**0.7초**).
3.  **리스트 컴프리헨션(List Comprehension):** `[i for i in range(1000)]`. 파이썬 내부 최적화가 적용되어 더 빠릅니다 (**0.3초**).
4.  **`list(range())` 형변환:** 가장 권장되는 방법으로, 미리 크기를 알고 한 번에 생성하기 때문에 압도적으로 빠릅니다 (**0.1초**).

---

### 💡 요약 및 팁
*   파이썬 리스트는 주소를 저장하는 방식 덕분에 다양한 타입의 객체를 담을 수 있지만, C 배열보다는 약간의 간접 비용이 발생합니다.
*   `append`는 내부적으로 리사이징을 거치지만 평균적으로는 $O(1)$의 성능을 냅니다.
*   성능이 중요하다면 `append`를 반복하기보다는 **`list(range())`**나 **리스트 컴프리헨션**을 사용하세요.
